AWS CloudTrail

The complete audit log of every API call made in your AWS account — who did what, when, and from where. The backbone of security investigations and compliance.

📋 Management Events 📦 Data Events 🔍 Insights 🏢 Org Trails 🔒 Log Integrity 🔎 Lake ⚡ EventBridge 📊 Athena Queries

🏗️ What is AWS CloudTrail?

Records every API call made in your AWS account — across the console, CLI, SDKs, and other AWS services.

One-liner CloudTrail = the audit log of who did what, when, from where, and with what result across your entire AWS account. Enabled by default for the last 90 days — create a Trail for permanent storage and advanced features.
📋 Event History (Default)

Last 90 days of management events, automatically recorded, no setup needed. Free. Searchable in the console. No S3 storage — view-only in console/CLI.

🛤️ Trail (Configured)

Continuous delivery of events to an S3 bucket — permanent retention. Required for events older than 90 days, CloudWatch Logs integration, and data events.

🔎 CloudTrail Lake

Managed data lake for CloudTrail events. SQL queries directly on events — no Athena setup needed. Retain up to 7 years. Newer, simpler alternative to S3 + Athena.

What Gets Recorded

Every CloudTrail event captures: Who (IAM user/role/root) · What action (API call, e.g. RunInstances) · When (timestamp) · Where from (source IP) · Which resource · Request parameters · Response elements · Whether it succeeded or was denied

📋 CloudTrail Event Types

Three categories of events — each with different cost, volume, and use case.

📋 Management Events
Control plane operations
Operations on AWS resources themselves: creating an EC2 instance, deleting an S3 bucket, modifying an IAM policy, creating a VPC. Free for the first copy delivered to S3. Enabled by default in Event History.
📦 Data Events
Data plane operations
High-volume operations on data inside resources: S3 GetObject/PutObject, Lambda Invoke, DynamoDB PutItem. Charged per event — can be very high volume. Disabled by default.
🔍 Insights Events
Anomaly detection
Automatically detects unusual API call volume or error rate patterns using ML. Flags spikes like a sudden burst of RunInstances calls or IAM policy changes. Additional charge, opt-in per trail.

Read-Only vs Write Events

👁️ Read-Only Events

DescribeInstances, GetObject, ListBuckets — operations that don't modify resources. Useful for understanding access patterns but rarely the focus of security investigations.

✏️ Write Events

RunInstances, DeleteBucket, PutBucketPolicy, CreateUser — operations that modify resources or state. These are what security teams focus on for incident investigation.

🏢 Trails — Continuous Logging

Configure a Trail to deliver events to S3 permanently and unlock advanced features.

🌍 Single-Region Trail

Captures events only in the region where it's created. Rarely recommended — you'll miss activity in other regions. Use only if your workload is genuinely single-region.

🌐 All-Regions Trail (Recommended)

Captures events from all AWS regions, including future regions automatically. Single S3 bucket destination. The default and recommended choice for almost every account.

🏢 Organization Trail

Created in the AWS Organizations management account. Automatically applies to all member accounts. Member accounts cannot disable or modify it. Best for centralised governance.

👤 Account Trail

Created within a single AWS account. Visible and manageable only within that account (unless it's an org trail). Good for account-specific custom requirements alongside an org trail.

Trail Setup Checklist

  • 1
    Create TrailName it, choose "Apply to all regions" = Yes. For Organizations, check "Apply to my organization".
  • 2
    Choose S3 DestinationCreate a new bucket or use existing. Enable bucket versioning + Object Lock for tamper-evidence.
  • 3
    Enable Log File ValidationCreates SHA-256 digest files to detect tampering. Always enable for compliance trails.
  • 4
    Enable SSE-KMS EncryptionEncrypt log files with a customer-managed KMS key for audit trail of who accessed the logs themselves.
  • 5
    Send to CloudWatch Logs (optional)Enables real-time alerting via CloudWatch Alarms and Metric Filters on specific API calls.
  • 6
    Select Data Events (optional)Enable S3/Lambda/DynamoDB data events only if you need data-plane visibility — watch for cost.
AWS CLI Create an all-regions trail with log validation + KMS 
aws cloudtrail create-trail \
  --name org-audit-trail \
  --s3-bucket-name my-cloudtrail-logs-bucket \
  --is-multi-region-trail \
  --enable-log-file-validation \
  --kms-key-id arn:aws:kms:us-east-1:123456789012:key/abc-123 \
  --is-organization-trail

# Start logging
aws cloudtrail start-logging --name org-audit-trail

# Enable CloudWatch Logs delivery for real-time alerting
aws cloudtrail update-trail \
  --name org-audit-trail \
  --cloud-watch-logs-log-group-arn arn:aws:logs:us-east-1:123456789012:log-group:CloudTrail/DefaultLogGroup:* \
  --cloud-watch-logs-role-arn arn:aws:iam::123456789012:role/CloudTrail_CloudWatchLogs_Role

🔍 CloudTrail Insights & Lake

Advanced features for anomaly detection and large-scale event querying.

🔍 CloudTrail Insights

ML-based anomaly detection on management events. Establishes a baseline of normal API call volume, then flags deviations.

Detects: Sudden spike in RunInstances (possible compromised credentials launching crypto-mining instances). Unusual IAM policy change rate. Error rate spikes (possible misconfiguration or attack).

Opt-in per trail Additional cost

🔎 CloudTrail Lake

Managed data lake purpose-built for CloudTrail events. Run SQL queries directly — no need to set up Athena + Glue + S3 partitioning manually.

Retention: Up to 7 years. Sources: CloudTrail events, AWS Config configuration items, and custom application events via PutAuditEvents API.

No Athena setup 7yr retention

CloudTrail Lake — Sample Query

SQL CloudTrail Lake — find all IAM policy changes in last 7 days 
SELECT eventTime, userIdentity.arn, eventName, requestParameters
FROM $EVENT_DATA_STORE_ID
WHERE eventSource = 'iam.amazonaws.com'
  AND eventName IN ('PutUserPolicy', 'AttachUserPolicy',
                    'PutRolePolicy', 'AttachRolePolicy',
                    'CreatePolicy', 'DeletePolicy')
  AND eventTime > date_add('day', -7, now())
ORDER BY eventTime DESC;

🔒 Log Integrity & Tamper Protection

An audit log is worthless if it can be modified. CloudTrail provides cryptographic proof of integrity.

Log File Validation Every hour, CloudTrail creates a digest file containing SHA-256 hashes of all log files delivered in that period, plus the hash of the previous digest file — forming a hash chain. Any tampering with a log file breaks the chain and is detectable.
🔗 Digest File Hash Chain

Each digest file references the previous one's hash. Validate with aws cloudtrail validate-logs. If any log file was modified or deleted, validation fails and identifies exactly which file.

🔐 SSE-KMS Encryption

Encrypt log files with a customer-managed KMS key. Only principals with kms:Decrypt permission can read logs — adds an access control layer beyond S3 bucket policy.

🔒 S3 Object Lock (WORM)

Apply Object Lock in Compliance mode to the CloudTrail S3 bucket. Makes log files immutable — cannot be deleted or modified by anyone, including root, until the retention period expires.

🚫 Restrictive Bucket Policy

Deny s3:DeleteObject and s3:PutObject (except from CloudTrail service) on the log bucket. Use SCPs to prevent member accounts from disabling the trail or modifying the bucket policy.

AWS CLI Validate log file integrity 
aws cloudtrail validate-logs \
  --trail-arn arn:aws:cloudtrail:us-east-1:123456789012:trail/org-audit-trail \
  --start-time 2026-06-01T00:00:00Z \
  --end-time 2026-06-18T00:00:00Z

# Output shows:
# Results requested for 2026-06-01T00:00:00Z to 2026-06-18T00:00:00Z
# Results found for the requested time range:

⚠️ Protecting Against Insider Threats A privileged user could disable CloudTrail to hide their tracks. Mitigate with: SCPs denying cloudtrail:StopLogging and cloudtrail:DeleteTrail org-wide, an EventBridge rule alerting on those API calls, and an Organization Trail that member accounts cannot touch at all.

C# Search CloudTrail Event History 
using Amazon.CloudTrail;
using Amazon.CloudTrail.Model;

var client = new AmazonCloudTrailClient();

// Find all DeleteBucket events in the last 24 hours
var response = await client.LookupEventsAsync(new LookupEventsRequest
{
    StartTime    = DateTime.UtcNow.AddHours(-24),
    EndTime      = DateTime.UtcNow,
    LookupAttributes =
    [
        new LookupAttribute
        {
            AttributeKey   = LookupAttributeKey.EventName,
            AttributeValue = "DeleteBucket"
        }
    ],
    MaxResults = 50
});

foreach (var evt in response.Events)
{
    Console.WriteLine($"{evt.EventTime}: {evt.Username} called {evt.EventName}");
    Console.WriteLine($"  Resources: {string.Join(\", \", evt.Resources.Select(r => r.ResourceName))}");
}
C# Create and configure a multi-region trail 
var createResponse = await client.CreateTrailAsync(new CreateTrailRequest
{
    Name                      = "app-audit-trail",
    S3BucketName              = "my-cloudtrail-logs-bucket",
    IsMultiRegionTrail        = true,
    EnableLogFileValidation   = true,
    KmsKeyId                  = "arn:aws:kms:us-east-1:123456789012:key/abc-123",
    CloudWatchLogsLogGroupArn = "arn:aws:logs:us-east-1:123456789012:log-group:CloudTrail/Default:*",
    CloudWatchLogsRoleArn     = "arn:aws:iam::123456789012:role/CloudTrail_CWLogs_Role"
});

// Start logging
await client.StartLoggingAsync(new StartLoggingRequest
    { Name = "app-audit-trail" });

// Add data events for a specific S3 bucket
await client.PutEventSelectorsAsync(new PutEventSelectorsRequest
{
    TrailName     = "app-audit-trail",
    EventSelectors =
    [
        new EventSelector
        {
            ReadWriteType           = ReadWriteType.All,
            IncludeManagementEvents = true,
            DataResources =
            [
                new DataResource
                {
                    Type   = "AWS::S3::Object",
                    Values = ["arn:aws:s3:::my-sensitive-bucket/"]
                }
            ]
        }
    ]
});
C# Query CloudTrail Lake event data store 
var queryResponse = await client.StartQueryAsync(new StartQueryRequest
{
    QueryStatement = $@"
        SELECT eventTime, userIdentity.arn, eventName, sourceIPAddress
        FROM {eventDataStoreId}
        WHERE eventName = 'ConsoleLogin'
          AND errorMessage = 'Failed authentication'
          AND eventTime > date_add('hour', -1, now())
        ORDER BY eventTime DESC"
});

// Poll for query results
var results = await client.GetQueryResultsAsync(new GetQueryResultsRequest
    { QueryId = queryResponse.QueryId, EventDataStore = eventDataStoreId });

foreach (var row in results.QueryResultRows)
    Console.WriteLine(string.Join(" | ", row.Select(c => c.Values.FirstOrDefault().Value)));

// Use case: detect brute-force login attempts in the last hour

📊 Querying CloudTrail Logs

Three ways to search and analyse your CloudTrail data — pick based on volume and retention needs.

🔍 Event History (Console)

Built-in console search. Last 90 days. Filter by event name, user, resource, time range. No setup. Good for quick lookups.

📊 Athena + S3

Query historical logs in S3 with standard SQL. Set up Glue table partition projection for performance. Pay per query (per TB scanned).

🔎 CloudTrail Lake

Purpose-built SQL query engine for CloudTrail. No Athena/Glue setup. Up to 7yr retention. Slightly higher cost per GB ingested but zero setup overhead.

Athena Table Setup

SQL Create Athena table over CloudTrail S3 logs 
CREATE EXTERNAL TABLE cloudtrail_logs (
  eventVersion STRING,
  userIdentity STRUCT<
    type: STRING, principalId: STRING,
    arn: STRING, accountId: STRING,
    userName: STRING>,
  eventTime STRING,
  eventSource STRING,
  eventName STRING,
  awsRegion STRING,
  sourceIPAddress STRING,
  userAgent STRING,
  errorCode STRING,
  errorMessage STRING,
  requestParameters STRING,
  responseElements STRING,
  recipientAccountId STRING
)
PARTITIONED BY (region STRING, year STRING, month STRING, day STRING)
ROW FORMAT SERDE 'org.apache.hive.hcatalog.data.JsonSerDe'
LOCATION 's3://my-cloudtrail-logs-bucket/AWSLogs/123456789012/CloudTrail/';

Common Investigation Queries

SQL Athena — who made changes to security groups? 
-- Who modified security groups in the last 7 days?
SELECT eventtime, useridentity.arn, eventname, sourceipaddress
FROM cloudtrail_logs
WHERE eventsource = 'ec2.amazonaws.com'
  AND eventname LIKE '%SecurityGroup%'
  AND year='2026' AND month='06'
ORDER BY eventtime DESC;

-- Find all console logins without MFA
SELECT eventtime, useridentity.username, sourceipaddress
FROM cloudtrail_logs
WHERE eventname = 'ConsoleLogin'
  AND json_extract_scalar(responseelements, '$.ConsoleLogin') = 'Success'
  AND json_extract_scalar(additionaleventdata, '$.MFAUsed') = 'No';

-- Failed API calls due to AccessDenied (potential probing)
SELECT useridentity.arn, eventname, COUNT(*) as attempts
FROM cloudtrail_logs
WHERE errorcode = 'AccessDenied'
GROUP BY useridentity.arn, eventname
ORDER BY attempts DESC;

💡 Best Practices

Production-grade CloudTrail setup for security, compliance, and operational visibility.

✅ Always Use an Organization Trail

Create one trail at the AWS Organizations management account level covering all member accounts. Member accounts cannot disable it — closes the insider-threat gap of an individual account turning off logging.

✅ Enable in All Regions

Always select "Apply trail to all regions". Attackers often target less-monitored regions specifically because teams forget to enable logging there.

✅ Enable Log File Validation

Always turn on log file validation. Costs nothing extra and gives you cryptographic proof that logs haven't been tampered with — critical for compliance and forensic investigations.

✅ Encrypt with Customer-Managed KMS

Use a CMK instead of the default. Gives you an audit trail (via the key's own CloudTrail events) of exactly who decrypted/read the audit logs themselves.

✅ Lock Down the Log Bucket

Apply a restrictive bucket policy denying delete/modify except from the CloudTrail service. Add S3 Object Lock in Compliance mode for true immutability. Block public access.

✅ Alert on Critical Events in Real-Time

Send to CloudWatch Logs and create Metric Filters + Alarms for: StopLogging, DeleteTrail, root account usage, CreateUser/AttachUserPolicy with admin policies, failed console logins.

✅ Use SCPs to Prevent Tampering

Deny cloudtrail:StopLogging, cloudtrail:DeleteTrail, and cloudtrail:UpdateTrail via Service Control Policy at the OU level — even account admins cannot disable the org trail.

✅ Enable Data Events Selectively

Don't enable S3/Lambda data events account-wide by default — costly at scale. Enable only for specific sensitive buckets/functions where data-plane visibility is genuinely needed.

🏁 Summary

AWS CloudTrail — the complete picture at a glance.

📋 Event Types

Management (control plane, default-on, free) · Data (data plane, high volume, paid) · Insights (ML anomaly detection, opt-in).

🏢 Trails

Event History = last 90 days, free, console only. Trail = permanent S3 delivery. Organization Trail = applies to all member accounts, cannot be disabled by them.

🔎 Querying

Console search (90 days) → Athena + S3 (historical, pay per query) → CloudTrail Lake (managed SQL, up to 7yr retention, no setup).

🔒 Integrity

Log file validation = SHA-256 hash chain digest files. SSE-KMS encryption. S3 Object Lock for WORM immutability. SCPs prevent trail tampering.

⚡ Real-Time Alerting

CloudTrail → CloudWatch Logs → Metric Filter → Alarm → SNS. Or CloudTrail → EventBridge rule → Lambda/SNS for specific event patterns.

🧩 SDK

LookupEventsAsync for Event History search. CreateTrailAsync + PutEventSelectorsAsync for trail management. StartQueryAsync for CloudTrail Lake SQL.
