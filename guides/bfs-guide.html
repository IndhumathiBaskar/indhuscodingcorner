🌊 What is Breadth-First Search?

Starting from zero. No jargon. Just plain understanding — and the analogy that makes it click forever.

💡 BFS in One Sentence

BFS is a way to explore a structure (tree or graph) by visiting all neighbours at the current level first, before going deeper. It explores wide before going deep.

🌊 The Water Ripple Analogy

💧Drop a stone in a pond

When you drop a stone into still water, ripples spread outward in circles. All points at distance 1 are touched first, then all points at distance 2, then distance 3. The water doesn't shoot in one direction — it spreads uniformly in all directions at the same time.

That is BFS. Start at a node. Visit all its neighbours (distance 1). Then all their neighbours (distance 2). Then distance 3. Layer by layer, level by level.

🔥Wildfire spreading through a forest

A fire starts at one tree. It immediately spreads to all adjacent trees at the same time. Then from all those trees, it spreads one more step outward simultaneously. This "wave expansion" is BFS — every node at the current frontier is processed before moving to the next frontier.

This is also why BFS naturally finds the shortest path — the fire reaches the nearest trees first, then farther ones.

🆚 BFS vs DFS — The Core Difference

🌊 BFS — Breadth First (Wide then Deep)

  • Explore all nodes at level 1 before going to level 2
  • Uses a Queue (First In, First Out — FIFO)
  • Like reading a book page by page, left to right
  • Naturally finds the shortest path (unweighted)
  • Good for: nearest neighbour, level-order, shortest path

🌲 DFS — Depth First (Deep then Wide)

  • Follow one path all the way to the end, then backtrack
  • Uses a Stack / Recursion (Last In, First Out — LIFO)
  • Like diving into a rabbit hole until you hit the bottom
  • Does NOT guarantee shortest path
  • Good for: cycle detection, paths, puzzles, tree traversal

📌 The Critical Difference from DFS

DFS uses a Stack (go deep, remember where you came from). BFS uses a Queue (process in order of arrival, level by level). Swapping stack for queue in the DFS code is literally all you need — and the behaviour completely changes.

🧱 Core Concepts — Queue, Levels, and FIFO

Part 1: The Queue — BFS's Engine

🍽️Restaurant queue analogy

Imagine people waiting in a queue to enter a restaurant. The first person who arrived is the first to enter. New people join at the back. This is FIFO — First In, First Out. BFS uses exactly this: the first node added to the queue is the first one processed. This is what keeps exploration level by level.

📋 Queue vs Stack — The Key Structural Difference
📦 Queue (BFS)
  • Add to BACK (Enqueue)
  • Remove from FRONT (Dequeue)
  • First In = First Out (FIFO)
  • Processes nodes in arrival order
  • C#: Queue<T> or LinkedList<T>
📚 Stack (DFS)
  • Add to TOP (Push)
  • Remove from TOP (Pop)
  • Last In = First Out (LIFO)
  • Processes most recent node first
  • C#: Stack<T> or recursion

Part 2: Level-by-Level Exploration

BFS on a tree naturally visits nodes level by level. Level 0 = root. Level 1 = root's children. Level 2 = grandchildren. And so on.

🌊 BFS Level-Order on a Tree
BFS Order: A → B → C → D → E → F → G

Part 3: Why Queue = Level Order (the intuition)

1

Enqueue root (A)

Queue: [A]. A is at level 0.

2

Dequeue A → enqueue its children (B, C)

Queue: [B, C]. B and C are level 1 — added after A.

3

Dequeue B → enqueue B's children (D, E)

Queue: [C, D, E]. C is still there from level 1. D, E join after it.

4

Dequeue C → enqueue C's children (F, G)

Queue: [D, E, F, G]. All level 1 nodes done. All level 2 queued up.

5

Process D, E, F, G one by one

Queue empties. BFS complete. The FIFO property naturally enforced level order!

✅ The Insight

Because FIFO processes nodes in the order they were added — and children are always added after their parent's entire level — the queue automatically enforces level-by-level processing. No extra logic needed.

🤔 Why BFS? When Should You Use It?

💡 BFS's Superpower

BFS guarantees the shortest path (in terms of number of edges) in an unweighted graph. Because it expands outward level by level, the first time it reaches a node, it has taken the fewest steps to get there.

⚠️ Memory Warning for Wide Graphs

In a graph where one node connects to 1,000,000 others (e.g. a popular social media user), BFS will enqueue all 1 million neighbours at once. The queue can explode in size. DFS would only hold the current path. Choose wisely based on graph shape.

🌍 Real-World BFS — Where You See It Daily

🗺️Google Maps — Find Nearest Petrol StationClassic BFS use case

When Maps finds the nearest petrol station, it starts at your current location and expands outward — checking all roads 1 junction away, then 2, then 3. The first petrol station it encounters is the closest one. That's BFS shortest path in action.

Weighted version (actual road times): Uses Dijkstra's algorithm — which is essentially BFS extended to handle edge weights.

🤝LinkedIn — Degrees of Connection"You're 2nd degree connected"

LinkedIn's "2nd connection" and "3rd+ connection" labels are BFS levels. Start at your profile. Level 1 = your direct connections. Level 2 = their connections (who aren't yours). Level 3 = one more hop out. BFS tells LinkedIn exactly how many hops apart two people are.

Also: "People You May Know" — BFS finds people at distance 2 (friends of friends) and recommends them.

🎨Paint Bucket Tool — MS Paint / PhotoshopFlood fill = BFS

When you click the paint bucket on a white area in MS Paint, it fills all connected pixels of the same colour. It starts at the clicked pixel, enqueues all 4 adjacent same-colour pixels, processes them, enqueues their neighbours — expanding outward until it hits a different colour boundary. Pure BFS on a 2D grid.

📡Network Broadcasting (LAN Discovery)AWS VPC / peer discovery

When a device joins a network and needs to discover all other devices, it broadcasts a message to immediate neighbours. Each device that receives it forwards to its neighbours. This wave-propagation is BFS. In AWS, Service Discovery (ECS, App Mesh) uses similar broadcast-and-discover patterns — finding all healthy instances reachable from a given point.

🎮Game AI — Shortest Path for EnemyGrid-based pathfinding

In grid-based games (like Chess, Pac-Man, strategy games), enemy AI uses BFS to find the shortest route to the player. The game map is a graph (cells = nodes, walls = no edges). BFS from the enemy's position finds the minimum steps to reach the player.

A* algorithm (used in most modern games) is BFS + a heuristic to make it faster when you know the target direction.

🏗️Dependency Level Resolution (Kahn's Algorithm)NuGet / npm build order

Kahn's algorithm for topological sort is BFS-based. It processes all packages with zero dependencies first (level 0), then packages whose dependencies are all resolved (level 1), and so on. MSBuild and NuGet restore can use this approach to determine build parallelism — packages at the same BFS level can be built simultaneously.

🌐Web Crawler — Breadth StrategyIndex nearest pages first

A BFS crawler visits all links on the current page before following those links deeper. This keeps the crawler close to the starting domain first — useful when you want complete coverage of one site before exploring outbound links. Google's PageRank algorithm benefits from BFS-crawled data to understand link structure at each "hop level".

🏥Hospital Contact TracingCOVID-19 exposure chains

During pandemic tracing, health authorities start from one confirmed case. BFS finds all direct contacts (level 1), then contacts of contacts (level 2). The level number tells you the degree of exposure risk. Level 1 = quarantine immediately. Level 2 = test and monitor. This is BFS on a contact graph.

⚙️ How BFS Works — Step by Step

BFS Algorithm — Plain English

📋 BFS Algorithm Flow
🟢 Start: Enqueue the root/start node. Mark it visited.
  • Add start to the Queue
  • Add start to visited set
↓
🔁 Loop: While the queue is not empty...
  • Dequeue the front node (current)
  • Process it (print, check, store)
↓
👀 Look at current node's neighbours
  • For each neighbour: if NOT visited → mark visited → enqueue it
  • If already visited → skip (prevents cycles)
↓
↩️ Repeat until Queue is empty
  • Empty queue = all reachable nodes have been visited
↓
✅ Done! All nodes processed level by level.

Shortest Path — How BFS tracks it

BFS naturally records distance from the source. The first time you reach a node, that's the shortest distance. Track it with a dictionary:

📏 Distance tracking during BFS
—
dist[A] = 0
Start node. Distance = 0.
—
dist[B] = 1
B is a neighbour of A. dist[A] + 1 = 1.
—
dist[C] = 1
C is a neighbour of A. dist[A] + 1 = 1.
—
dist[D] = 2
D is a neighbour of B. dist[B] + 1 = 2.
—
dist[E] = 2
E is a neighbour of B. dist[B] + 1 = 2.
—
dist[F] = 2
F is a neighbour of C. dist[C] + 1 = 2.

🔑 Key Insight: Why BFS = Shortest Path

Because BFS visits all nodes at distance k before any node at distance k+1, the first time it reaches a node is guaranteed to be via the shortest path. If a shorter path existed, BFS would have found it in an earlier level.

📝 Pseudocode — Every BFS Pattern

Pseudocode bridges the gap between concept and real code. Learn these patterns and BFS becomes second nature.

📌 How to read the pseudocode

Pseudocode uses natural English + common programming constructs. It's not tied to any language. You'll see: ENQUEUE = add to back of queue, DEQUEUE = remove from front, for each = loop over, ← = assign value. Words in CAPS are operations.

① Core BFS — The Universal Template

pseudocode — bfs core template 
BFS(graph, start):
    // Set up
    queue    ← new Queue
    visited  ← new Set

    // Initialise with start node
    ENQUEUE(queue, start)
    ADD(visited, start)

    while queue is not empty:
        current ← DEQUEUE(queue)          // take from front
        PROCESS(current)                  // do something with it

        for each neighbour in NEIGHBOURS(current):
            if neighbour not in visited:
                ADD(visited, neighbour)   // mark BEFORE enqueue!
                ENQUEUE(queue, neighbour)

    // All reachable nodes have been visited

⚠️ Mark visited BEFORE enqueue, not after dequeue

A common bug: marking nodes as visited when you dequeue them instead of when you enqueue them. This lets the same node get enqueued multiple times (once from each neighbour that sees it). Always mark visited at enqueue time.

② BFS on a Tree (No visited set needed)

pseudocode — bfs level-order traversal on tree 
BFS_TREE(root):
    if root is null: return

    queue ← new Queue
    ENQUEUE(queue, root)

    while queue is not empty:
        current ← DEQUEUE(queue)
        PROCESS(current)                  // print / store value

        if current.left  is not null: ENQUEUE(queue, current.left)
        if current.right is not null: ENQUEUE(queue, current.right)

    // Trees have no cycles → no visited set needed

③ BFS with Level Tracking

pseudocode — bfs tracking which level each node is on 
BFS_LEVELS(root):
    if root is null: return

    queue  ← new Queue
    result ← new List of Lists        // one inner list per level

    ENQUEUE(queue, root)

    while queue is not empty:
        levelSize ← SIZE(queue)          // nodes at THIS level
        currentLevel ← new List

        repeat levelSize times:
            node ← DEQUEUE(queue)
            ADD(currentLevel, node.value)

            if node.left  is not null: ENQUEUE(queue, node.left)
            if node.right is not null: ENQUEUE(queue, node.right)

        ADD(result, currentLevel)        // save this level's values

    return result
    // result[0] = [A], result[1] = [B, C], result[2] = [D, E, F, G]

④ BFS Shortest Path — with Distance Tracking

pseudocode — bfs shortest path (unweighted graph) 
BFS_SHORTEST_PATH(graph, start, target):
    queue    ← new Queue
    visited  ← new Set
    dist     ← new Map                  // node → distance from start
    parent   ← new Map                  // node → how we got here

    ENQUEUE(queue, start)
    ADD(visited, start)
    dist[start] ← 0
    parent[start] ← null

    while queue is not empty:
        current ← DEQUEUE(queue)

        if current == target:
            return RECONSTRUCT_PATH(parent, start, target)

        for each neighbour in NEIGHBOURS(current):
            if neighbour not in visited:
                ADD(visited, neighbour)
                dist[neighbour] ← dist[current] + 1
                parent[neighbour] ← current
                ENQUEUE(queue, neighbour)

    return null                          // no path found

RECONSTRUCT_PATH(parent, start, target):
    path ← new List
    node ← target
    while node is not null:
        ADD_FRONT(path, node)
        node ← parent[node]
    return path                          // [start, ..., target]

⑤ BFS on a 2D Grid (Flood Fill / Paint Bucket)

pseudocode — bfs flood fill on a 2d grid 
BFS_GRID(grid, startRow, startCol):
    rows    ← HEIGHT(grid)
    cols    ← WIDTH(grid)
    queue   ← new Queue
    visited ← 2D boolean grid of same size, all false

    directions ← [(0,1), (0,-1), (1,0), (-1,0)]   // right, left, down, up

        (row, col) ← DEQUEUE(queue)

⑥ BFS — Check if Path Exists

pseudocode — bfs reachability check 
BFS_REACHABLE(graph, start, target):
    queue   ← new Queue
    visited ← new Set

    ENQUEUE(queue, start)
    ADD(visited, start)

    while queue is not empty:
        current ← DEQUEUE(queue)

        if current == target:
            return true               // found it!

        for each neighbour in NEIGHBOURS(current):
            if neighbour not in visited:
                ADD(visited, neighbour)
                ENQUEUE(queue, neighbour)

    return false                      // target not reachable

⑦ Kahn's Algorithm — BFS-based Topological Sort

pseudocode — kahn's topological sort (bfs variant) 
KAHN_TOPO_SORT(graph, nodes):
    inDegree ← Map: node → count of incoming edges
    queue    ← new Queue
    result   ← new List

    // Initialise inDegree for all nodes
    for each node in nodes:
        inDegree[node] ← 0
    for each (u, v) in EDGES(graph):
        inDegree[v] ← inDegree[v] + 1

    // Enqueue nodes with no dependencies (inDegree = 0)
    for each node in nodes:
        if inDegree[node] == 0:
            ENQUEUE(queue, node)

    while queue is not empty:
        current ← DEQUEUE(queue)
        ADD(result, current)

        for each neighbour in NEIGHBOURS(current):
            inDegree[neighbour] ← inDegree[neighbour] - 1
            if inDegree[neighbour] == 0:
                ENQUEUE(queue, neighbour)    // ready to process

    if SIZE(result) != SIZE(nodes):
        return ERROR("Cycle detected!")     // can't topologically sort a cycle

    return result

🧠 Pattern Recognition

Every BFS pseudocode follows the same skeleton: init queue → enqueue start → while queue not empty → dequeue → process → enqueue unvisited neighbours. The only variation is what extra data you track (distance, level, parent, inDegree). Recognise the skeleton and you can write any BFS variant.

2. BFS Level-Order Traversal (Tree)

c# — bfs tree traversal using queue 
// Visit tree nodes level by level (BFS = Level-order traversal)
public static List<int> LevelOrder(TreeNode? root)
{
    var result = new List<int>();
    if (root == null) return result;

    var queue = new Queue<TreeNode>();
    queue.Enqueue(root);

    while (queue.Count > 0)
    {
        var node = queue.Dequeue();     // take from front (FIFO)
        result.Add(node.Val);

        if (node.Left  != null) queue.Enqueue(node.Left);
        if (node.Right != null) queue.Enqueue(node.Right);
    }

    return result;
    // For tree: A,B,C,D,E,F,G → returns [A,B,C,D,E,F,G]
}

3. Level-by-Level Groups (Nested List)

c# — bfs collecting nodes per level 
// Returns [[A], [B,C], [D,E,F,G]] — one list per level
public static List<List<int>> LevelOrderGroups(TreeNode? root)
{
    var result = new List<List<int>>();
    if (root == null) return result;

    var queue = new Queue<TreeNode>();
    queue.Enqueue(root);

    while (queue.Count > 0)
    {
        int levelSize = queue.Count;     // nodes at THIS level
        var level = new List<int>();

        for (int i = 0; i < levelSize; i++)
        {
            var node = queue.Dequeue();
            level.Add(node.Val);

            if (node.Left  != null) queue.Enqueue(node.Left);
            if (node.Right != null) queue.Enqueue(node.Right);
        }

        result.Add(level);             // save this level
    }

    return result;
}

4. BFS on a Graph (with Visited Set)

c# — graph bfs using adjacency list 
// Graph traversal — must track visited to handle cycles
public class GraphBFS
{
    private Dictionary<string, List<string>> _graph = new()
    {
        { "EntryAPI",   new() { "AuthService", "DataService" } },
        { "AuthService", new() { "TokenStore" } },
        { "DataService", new() { "Database", "Cache" } },
        { "TokenStore",  new() },
        { "Database",   new() },
        { "Cache",      new() }
    };

    public void BFS(string start)
    {
        var queue   = new Queue<string>();
        var visited = new HashSet<string>();

        queue.Enqueue(start);
        visited.Add(start);              // mark at enqueue time!

        while (queue.Count > 0)
        {
            var current = queue.Dequeue();
            Console.WriteLine($"Visiting: {current}");

            foreach (var neighbour in _graph[current])
            {
                if (!visited.Contains(neighbour))
                {
                    visited.Add(neighbour);
                    queue.Enqueue(neighbour);
                }
            }
        }
    }
}
// Output: EntryAPI → AuthService → DataService → TokenStore → Database → Cache

5. Shortest Path — BFS with Distance Map

c# — bfs shortest path with path reconstruction 
public static List<string>? ShortestPath(
    Dictionary<string, List<string>> graph,
    string start,
    string target)
{
    var queue   = new Queue<string>();
    var visited = new HashSet<string>();
    var parent  = new Dictionary<string, string?>();  // track path

    queue.Enqueue(start);
    visited.Add(start);
    parent[start] = null;

    while (queue.Count > 0)
    {
        var current = queue.Dequeue();

        if (current == target)
            return ReconstructPath(parent, start, target);

        foreach (var neighbour in graph[current])
        {
            if (!visited.Contains(neighbour))
            {
                visited.Add(neighbour);
                parent[neighbour] = current;       // remember where we came from
                queue.Enqueue(neighbour);
            }
        }
    }

    return null;  // no path exists
}

private static List<string> ReconstructPath(
    Dictionary<string, string?> parent,
    string start, string target)
{
    var path = new LinkedList<string>();
    var current = (string?)target;

    while (current != null)
    {
        path.AddFirst(current);           // build path backwards
        current = parent[current];
    }

    return path.ToList();
}
// Returns: ["EntryAPI", "DataService", "Database"]

6. Flood Fill on a 2D Grid (Paint Bucket)

c# — bfs flood fill algorithm 
// Classic MS Paint bucket fill — BFS on a 2D grid
public static void FloodFill(int[,] grid, int row, int col, int newColour)
{
    int rows = grid.GetLength(0);
    int cols = grid.GetLength(1);
    int oldColour = grid[row, col];

    if (oldColour == newColour) return;

    var queue   = new Queue<(int, int)>();
    var visited = new HashSet<(int, int)>();
    int[] dr = { 0, 0, 1, -1 };  // right, left, down, up
    int[] dc = { 1, -1, 0, 0 };

    queue.Enqueue((row, col));
    visited.Add((row, col));

    while (queue.Count > 0)
    {
        var (r, c) = queue.Dequeue();
        grid[r, c] = newColour;              // fill this cell

        for (int d = 0; d < 4; d++)
        {
            int nr = r + dr[d], nc = c + dc[d];

            if (nr >= 0 && nr < rows &&
                nc >= 0 && nc < cols &&
                !visited.Contains((nr, nc)) &&
                grid[nr, nc] == oldColour)
            {
                visited.Add((nr, nc));
                queue.Enqueue((nr, nc));
            }
        }
    }
}

7. Kahn's Algorithm — BFS Topological Sort

c# — kahn's topological sort (bfs-based) 
// Use case: determine C# project build order by dependency
public static List<string> TopologicalSort(
    Dictionary<string, List<string>> graph)
{
    var inDegree = new Dictionary<string, int>();
    var result   = new List<string>();

    // Init in-degrees
    foreach (var node in graph.Keys) inDegree[node] = 0;
    foreach (var (_, neighbours) in graph)
        foreach (var n in neighbours) inDegree[n]++;

    // Enqueue nodes with no dependencies
    var queue = new Queue<string>(
        inDegree.Where(kv => kv.Value == 0).Select(kv => kv.Key));

    while (queue.Count > 0)
    {
        var current = queue.Dequeue();
        result.Add(current);

        foreach (var neighbour in graph[current])
        {
            inDegree[neighbour]--;
            if (inDegree[neighbour] == 0)
                queue.Enqueue(neighbour);   // all deps resolved
        }
    }

    if (result.Count != graph.Count)
        throw new InvalidOperationException("Cycle detected!");

    return result;
}

// ======= TREE BFS (no visited needed) =======
var q = new Queue<TreeNode>();
q.Enqueue(root);
while (q.Count > 0) {
    var node = q.Dequeue();
    // process node
    if (node.Left  != null) q.Enqueue(node.Left);
    if (node.Right != null) q.Enqueue(node.Right);
}

// ======= GRAPH BFS (need visited) =======
var q = new Queue<string>();
var seen = new HashSet<string>();
q.Enqueue(start); seen.Add(start);      // mark at ENQUEUE time
while (q.Count > 0) {
    var curr = q.Dequeue();
    // process curr
    foreach (var nb in graph[curr])
        if (seen.Add(nb)) q.Enqueue(nb);   // HashSet.Add returns false if exists
}

// ======= LEVEL TRACKING =======
int level = 0;
while (q.Count > 0) {
    int size = q.Count;                  // snapshot of THIS level
    for (int i = 0; i < size; i++) {
        var node = q.Dequeue();
        // process at level {level}
        // enqueue children...
    }
    level++;
}

Complexity Summary

⏱️ Time Complexity

  • Tree BFS: O(n) — every node visited once
  • Graph BFS: O(V + E) — vertices + edges
  • Grid BFS: O(rows × cols) — every cell once
  • Kahn's: O(V + E)

💾 Space Complexity

  • Queue: O(w) — max width of tree/graph
  • Visited set: O(V)
  • Worst case (complete binary tree): O(n/2) = O(n)
  • Grid: O(rows × cols)

Common Mistakes

  • Marking visited at DEQUEUE instead of ENQUEUE → same node enqueued multiple times
  • Not initialising visited set for graphs → infinite loop on cycles
  • Forgetting to check for null root before enqueuing → NullReferenceException
  • Using BFS for weighted shortest path → wrong answers (use Dijkstra)
  • Taking levelSize snapshot AFTER enqueuing children → wrong level count
  • Using a Stack instead of Queue → accidentally doing DFS

Signs You Need BFS

  • Problem asks for "shortest path" or "minimum steps" (unweighted)
  • Problem asks for "nearest" node with some property
  • You need to process nodes "level by level" or "layer by layer"
  • Problem involves spreading/propagation (fire, infection, flood)
  • You want ALL nodes at distance k before distance k+1
  • You need to check if a path exists (reachability)
  • Grid problems: find minimum steps to reach a cell
  • Social graph: degrees of separation
