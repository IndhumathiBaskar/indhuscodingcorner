📦 What is AWS CodeArtifact?

AWS CodeArtifact is a fully managed artifact repository service. It stores, publishes, and shares software packages — NuGet, npm, PyPI, Maven, Gradle, Swift, and generic binaries. It acts as a private proxy in front of public registries (nuget.org, npmjs.com, pypi.org) so your CI/CD pipelines never need direct internet access to pull dependencies.

💡 The Core Mental Model

Think of CodeArtifact as your company's private package registry. Your .NET projects reference it instead of nuget.org. When they request a package, CodeArtifact checks its local cache first — instant. If not cached, it fetches from nuget.org, stores a copy, and returns it. Future requests are served from cache. Your internal NuGet packages (shared libraries, utilities) are published directly here and never go to the public internet.

Key Concepts

🏢

Domain

Top-level container. Groups all repositories for an org. One domain per company.

🗄️

Repository

Stores packages of one format (NuGet, npm, etc.). A domain can have many repos.

🔗

Upstream Source

Public registry (nuget.org) or another repo that packages are fetched from if not cached locally.

📄

Package

A named artifact (e.g., WatsonShared) across all its versions.

🏷️

Package Version

A specific release (e.g., WatsonShared 1.2.3). Has its own assets, tags, and status.

🔑

Auth Token

Short-lived token (12h) generated via AWS CLI/SDK for NuGet/npm/pip authentication.

🔒

IAM Policy

Fine-grained access — read-only, read-write, or admin per repo or package.

📋

Package Origin

Control whether packages can be ingested from upstream or only published internally.

🌍

Cross-account

Share repos across AWS accounts via resource-based domain/repo policies.

Create Domain + Repos via AWS CLI

bash · Create Domain and Repositories 
# ── Step 1: Create the Domain ─────────────────────────────────
aws codeartifact create-domain \
  --domain watson-domain \
  --tags Team=Watson,CostCenter=Platform

# Get domain ARN
aws codeartifact describe-domain \
  --domain watson-domain \
  --query domain.arn \
  --output text

# ── Step 2: Create the proxy repo (nuget.org upstream) ────────
aws codeartifact create-repository \
  --domain        watson-domain \
  --repository    nuget-store \
  --description   "Proxy cache for nuget.org" \
  --upstreams     repositoryName=nuget-org-upstream

# Add nuget.org as an external connection to nuget-store
aws codeartifact associate-external-connection \
  --domain     watson-domain \
  --repository nuget-store \
  --external-connection "public:nuget-org"

# ── Step 3: Create the internal repo ──────────────────────────
aws codeartifact create-repository \
  --domain      watson-domain \
  --repository  watson-nuget \
  --description "Watson team private NuGet packages" \
  --upstreams   repositoryName=nuget-store

# ── List repositories ──────────────────────────────────────────
aws codeartifact list-repositories \
  --domain watson-domain

# ── Get the NuGet endpoint URL ─────────────────────────────────
aws codeartifact get-repository-endpoint \
  --domain     watson-domain \
  --repository watson-nuget \
  --format     nuget \
  --query repositoryEndpoint \
  --output text

🟦 NuGet Setup — .NET Configuration

Configuring .NET to use CodeArtifact as a NuGet source requires three things: getting an auth token, adding the feed to NuGet config, and pointing your projects at it.

Step 1 — Get an Auth Token

bash · Get CodeArtifact Auth Token (expires in 12 hours) # Get token and store in environment variable export CODEARTIFACT_TOKEN=$(aws codeartifact get-authorization-token \ --domain watson-domain \ --domain-owner 123456789012 \ --query authorizationToken \ --output text) echo "Token obtained: ${CODEARTIFACT_TOKEN:0:20}..." # Get the NuGet endpoint URL export NUGET_SOURCE=$(aws codeartifact get-repository-endpoint \ --domain watson-domain \ --repository watson-nuget \ --format nuget \ --query repositoryEndpoint \ --output text) echo "NuGet endpoint: $NUGET_SOURCE" # → https://watson-domain-123456789012.d.codeartifact.eu-west-1.amazonaws.com/nuget/watson-nuget/v3/index.json

Step 2 — Add to NuGet Config

bash · Add CodeArtifact as NuGet Source # Add the source (run once per machine) dotnet nuget add source \ "$NUGET_SOURCE" \ --name "watson-nuget" \ --username "aws" \ --password "$CODEARTIFACT_TOKEN" \ --store-password-in-clear-text # Verify sources dotnet nuget list source # Remove old source if needed dotnet nuget remove source "old-source-name" # Update the token when it expires (12h) dotnet nuget update source "watson-nuget" \ --username "aws" \ --password "$CODEARTIFACT_TOKEN" \ --store-password-in-clear-text

nuget.config — Per-Project or Global

XML · nuget.config — Check into your repo root <?xml version="1.0" encoding="utf-8"?> <configuration> <packageSources> <!-- Clear default sources so ONLY CodeArtifact is used --> <clear /> <!-- Watson internal + proxied packages --> <add key="watson-nuget" value="https://watson-domain-123456789012.d.codeartifact.eu-west-1.amazonaws.com/nuget/watson-nuget/v3/index.json" /> </packageSources> <packageSourceCredentials> <watson-nuget> <!-- Token injected from environment variable at runtime --> <add key="Username" value="aws" /> <add key="ClearTextPassword" value="%CODEARTIFACT_TOKEN%" /> </watson-nuget> </packageSourceCredentials> </configuration>

🔗 Upstream Sources — Package Resolution Chain

Upstream sources define where CodeArtifact looks for packages it doesn't have locally. When a package is requested, CodeArtifact checks its own cache first, then walks up the upstream chain.

Resolution Chain — How Package Lookup Works

🔗 Package Resolution Chain — watson-nuget
1️⃣ watson-nuget (local)

Check internal repo first — private packages (WatsonShared, WatsonCore) found here

↓ not found
2️⃣ nuget-store (upstream repo)

Check the proxy cache — previously fetched packages from nuget.org served from here

↓ not cached
3️⃣ nuget.org (external connection)

Fetch from public registry → store in nuget-store cache → return to developer

💡 Cache-Once, Serve-Forever

Once a package version is fetched from nuget.org and stored in your nuget-store, it's permanently cached. Even if nuget.org removes the package or goes offline, your builds keep working. This also means no package can be silently mutated under your feet — the stored version is immutable.

Upstream Priority Order

bash · Configure Upstream Priority # Update upstreams on watson-nuget (order matters!) # CodeArtifact searches upstreams in the order listed aws codeartifact update-repository \ --domain watson-domain \ --repository watson-nuget \ --upstreams \ repositoryName=shared-libs \ # Search shared libs first repositoryName=nuget-store # Then public proxy cache # Describe repository upstreams aws codeartifact describe-repository \ --domain watson-domain \ --repository watson-nuget \ --query repository.upstreams

Package Origin Control — Prevent Supply Chain Attacks

Package origin controls let you restrict whether a package can come from upstream (public) sources or must be published internally. This prevents dependency confusion attacks where an attacker publishes a malicious package with the same name as your internal package on nuget.org.

bash · Set Package Origin — Block External Versions of Internal Package # BLOCK upstream ingestion for WatsonShared # Only allow versions published directly to watson-nuget aws codeartifact put-package-origin-configuration \ --domain watson-domain \ --repository watson-nuget \ --format nuget \ --package WatsonShared \ --restrictions upstream=BLOCK,publish=ALLOW # Check current origin config for a package aws codeartifact describe-package \ --domain watson-domain \ --repository watson-nuget \ --format nuget \ --package WatsonShared \ --query package.originConfiguration

Retention Policy — Auto-Delete Old Versions

bash · Package Group Retention Policy # Set retention policy — keep only latest 5 versions of internal packages aws codeartifact put-package-origin-configuration \ --domain watson-domain \ --repository watson-nuget \ --format nuget \ --package WatsonShared \ --restrictions upstream=BLOCK,publish=ALLOW # List all versions of a package aws codeartifact list-package-versions \ --domain watson-domain \ --repository watson-nuget \ --format nuget \ --package WatsonShared \ --status Published # Delete a specific old version aws codeartifact delete-package-versions \ --domain watson-domain \ --repository watson-nuget \ --format nuget \ --package WatsonShared \ --versions 1.0.0 1.0.1 1.0.2

🔒 IAM & Authentication

CodeArtifact uses two-layer authentication: IAM controls access to the CodeArtifact API, and a short-lived auth token (derived from IAM) authenticates NuGet/npm/pip tool calls against the package endpoint.

CodeArtifact Service — Token, Publish, Search

C# · CodeArtifactService.cs using Amazon.CodeArtifact; using Amazon.CodeArtifact.Model; public class CodeArtifactService { private readonly IAmazonCodeArtifact _ca; private readonly string _domain; private readonly string _domainOwner; public CodeArtifactService( IAmazonCodeArtifact ca, string domain, string domainOwner) { _ca = ca; _domain = domain; _domainOwner = domainOwner; } // ── GET AUTH TOKEN ──────────────────────────────────── public async Task<string> GetTokenAsync( long durationSeconds = 43200, // 12 hours default CancellationToken ct = default) { var resp = await _ca.GetAuthorizationTokenAsync( new GetAuthorizationTokenRequest { Domain = _domain, DomainOwner = _domainOwner, DurationSeconds = durationSeconds }, ct); return resp.AuthorizationToken; } // ── GET REPOSITORY ENDPOINT ─────────────────────────── public async Task<string> GetEndpointAsync( string repository, string format = "nuget", CancellationToken ct = default) { var resp = await _ca.GetRepositoryEndpointAsync( new GetRepositoryEndpointRequest { Domain = _domain, DomainOwner = _domainOwner, Repository = repository, Format = format }, ct); return resp.RepositoryEndpoint; } // ── LIST PACKAGES ───────────────────────────────────── public async Task<List<PackageSummary>> ListPackagesAsync( string repository, string format = "nuget", string? namePrefix = null, CancellationToken ct = default) { var packages = new List<PackageSummary>(); string? nextToken = null; do { var request = new ListPackagesRequest { Domain = _domain, DomainOwner = _domainOwner, Repository = repository, Format = format, NextToken = nextToken }; if (namePrefix is not null) request.PackagePrefix = namePrefix; var resp = await _ca.ListPackagesAsync(request, ct); packages.AddRange(resp.Packages); nextToken = resp.NextToken; } while (nextToken is not null); return packages; } // ── LIST PACKAGE VERSIONS ───────────────────────────── public async Task<List<PackageVersionSummary>> ListVersionsAsync( string repository, string packageName, string format = "nuget", CancellationToken ct = default) { var resp = await _ca.ListPackageVersionsAsync( new ListPackageVersionsRequest { Domain = _domain, DomainOwner = _domainOwner, Repository = repository, Format = format, Package = packageName, SortBy = PackageVersionSortType.PUBLISHED_TIME, Status = PackageVersionStatus.Published }, ct); return resp.Versions; } // ── DELETE OLD PACKAGE VERSIONS ────────────────────── public async Task DeleteVersionsAsync( string repository, string packageName, List<string> versions, string format = "nuget", CancellationToken ct = default) => await _ca.DeletePackageVersionsAsync( new DeletePackageVersionsRequest { Domain = _domain, DomainOwner = _domainOwner, Repository = repository, Format = format, Package = packageName, Versions = versions }, ct); // ── CONVENIENCE: GET TOKEN + ENDPOINT ──────────────── public async Task<(string Token, string Endpoint)> GetNuGetCredentialsAsync( string repository, CancellationToken ct = default) { var tokenTask = GetTokenAsync(ct: ct); var endpointTask = GetEndpointAsync(repository, ct: ct); await Task.WhenAll(tokenTask, endpointTask); return (tokenTask.Result, endpointTask.Result); } }

Best Practices & Common Mistakes

  • Create a separate proxy repo (nuget-store) and internal repo (watson-nuget) — don't mix proxied and internal packages in one repo
  • Use Package Origin Controls on all internal package names — block upstream ingestion to prevent dependency confusion attacks
  • Always set --store-password-in-clear-text flag when adding source in Linux CI — required because Linux NuGet doesn't have a secure credential store
  • Refresh the token in CI before each build — tokens are valid for 12h but a long-running pipeline may expire mid-build
  • Grant least-privilege IAM — developers get read-only, CI/CD gets publish, admins get delete/manage
  • Use the nuget.config file in your repo with <clear /> — ensures all devs and CI use only CodeArtifact, never nuget.org directly
  • Tag package versions with ApprovedForProd — gives you a governance audit trail for compliance
  • Delete old package versions regularly — cached upstream packages count toward storage billing

⚠️ Top CodeArtifact Mistakes

❌ Publishing to the proxy repo (nuget-store) — this repo is read-only from a proxy perspective; publish only to watson-nuget
❌ Not setting up Package Origin Controls — leaves you vulnerable to dependency confusion attacks where public package names shadow your internal ones
❌ Hardcoding the auth token in CI — tokens expire in 12h; always generate fresh using aws codeartifact get-authorization-token at build start
❌ Missing sts:GetServiceBearerToken in IAM policy — GetAuthorizationToken silently fails without this permission
❌ Using the same repo for all teams — separate repos per team gives better access control, cost attribution, and retention policies
❌ Not using nuget.config <clear /> — NuGet may still fall back to nuget.org and pull packages, bypassing your controls
❌ Forgetting to associate external connection — upstream repo won't proxy nuget.org until associate-external-connection is called
