☁️ What is Amazon CloudFront?

Amazon CloudFront is AWS's global Content Delivery Network (CDN). It caches and delivers your content — static files, APIs, video, dynamic responses — from 600+ Points of Presence (PoPs) in 100+ cities worldwide, putting data physically closer to users and reducing latency from seconds to milliseconds.

💡 CloudFront in One Sentence

CloudFront = a global cache layer that sits in front of your origin (S3, ALB, EC2, API Gateway) and serves requests from the nearest edge location — so a user in Mumbai doesn't have to wait for a response from us-east-1.

The Problem CloudFront Solves

😰 Without CloudFront

  • Every request hits your origin server in one AWS region
  • User in Tokyo → us-east-1 → 150ms+ latency every request
  • Origin handles all traffic — high load, high cost
  • DDoS attacks hit your origin directly
  • No built-in HTTPS for S3 static sites
  • No way to serve different content based on location

😌 With CloudFront

  • Cached content served from nearest edge — <10ms typical
  • Origin only receives cache misses — 80-95% traffic reduction
  • Built-in DDoS protection + AWS Shield Standard free
  • Free HTTPS with ACM certificate — one click
  • Lambda@Edge / CloudFront Functions for edge logic
  • WAF integration for application-layer protection

🏠 Origins — Where CloudFront Fetches Content

An origin is the authoritative source of your content. CloudFront forwards cache-miss requests to the origin, caches the response, and serves it to future users. You can have multiple origins per distribution and route to them based on path patterns (behaviors).

Supported Origin Types

🗄️ Amazon S3 Origin

  • Best for: static websites, file downloads, media assets
  • Use OAC (Origin Access Control) — never make bucket public
  • S3 website endpoints support redirects and custom error pages
  • S3 REST API endpoints used for private content with OAC
  • Can use S3 in different region — CloudFront handles geo-routing

⚖️ Application Load Balancer (ALB)

  • Best for: dynamic apps, microservices behind ALB
  • Use custom header to restrict ALB to CloudFront only
  • Supports WebSocket connections through CloudFront
  • ALB must be internet-facing (CloudFront connects over internet)
  • Use sticky sessions via CloudFront forwarding cookies

🌐 API Gateway

  • Best for: serverless APIs, Lambda-backed endpoints
  • API Gateway has its own CloudFront — adding your own gives extra control
  • Can add WAF rules not available on API Gateway directly
  • Useful for custom domain + caching on top of API Gateway
  • HTTP API recommended over REST API for cost at high volume

🌍 Custom HTTP Origin

  • Any publicly accessible HTTP/HTTPS endpoint
  • On-premise servers, third-party APIs, EC2 instances directly
  • Set origin protocol policy: HTTP only / HTTPS only / Match viewer
  • Can set custom HTTP headers CloudFront adds on every origin request
  • Configure connection timeout and read timeout (1–60s)

Origin Groups — Failover

📋 Cache Behavior → Origin Group
  • Route a behavior to an Origin Group instead of a single origin
↓
🟢 Primary Origin
  • CloudFront tries this first
  • e.g. S3 bucket us-east-1
🔵 Secondary Origin
  • Failover if primary returns 5xx or times out
  • e.g. S3 bucket eu-west-1
↓
⚙️ Failover Trigger
  • Configure which HTTP status codes trigger failover: 500, 502, 503, 504
  • Also triggers on connection timeout to primary origin
  • CloudFront retries the request against secondary automatically

💡 Restricting ALB to CloudFront Only

Set a custom origin header in CloudFront (e.g. X-CloudFront-Secret: abc123). In your ALB listener rules, add a condition: only forward requests that contain this header. Any request bypassing CloudFront (direct to ALB) will be rejected. Rotate the secret periodically.

📋 Caching & Cache Behaviors

Cache behaviors are the heart of CloudFront configuration — they define which origin to use, how long to cache, what to forward to the origin, and what functions to run. Every distribution has a default behavior (*) and you can add path-specific behaviors that take priority.

Cache Behavior Priority

📥 Incoming Request: GET /api/users/123
↓
CloudFront matches path against behaviors in priority order (top = highest)
Priority 0 — /api/*
  • Origin: ALB (api.example.com)
  • TTL: 0 (no cache) · Forward all headers
  • ✅ Matches — use this behavior
↓
Priority 1 — /static/*
  • Origin: S3 · TTL: 86400s · Not evaluated (previous matched)
↓
Default — * (catch-all)
  • Evaluated last if nothing else matches

Cache Key — What Makes a Unique Cache Entry

🔑 Cache Key Components

The cache key determines when CloudFront serves a cached response vs fetches a new one. By default only the URL path is the cache key. You can add headers, cookies, and query strings — but every addition fragments the cache and reduces hit ratio.

ComponentDefaultWhen to IncludeImpact on cache
URL path✅ Always includedAlways—
Query strings❌ ExcludedInclude version= for cache bustingFragments cache per param value
Headers❌ ExcludedInclude Accept-Language for i18nFragments per header value
Cookies❌ ExcludedInclude session cookie for personalised contentFragments per cookie value
HTTP methodGET + HEAD cachedPOST/PUT never cached by design—

Cache Policies vs Origin Request Policies

📦 Cache Policy

  • Defines what goes INTO the cache key
  • Min/Max/Default TTL settings
  • Which headers/cookies/query strings make unique cache entries
  • Rule: only include what actually changes the response
  • AWS managed policies: CachingOptimized, CachingDisabled, CachingOptimizedForUncompressedObjects

📤 Origin Request Policy

  • Defines what CloudFront forwards to origin on cache miss
  • Separate from cache key — forward without caching per value
  • e.g. Forward Authorization header to origin but don't include in cache key
  • Forward viewer's IP via CloudFront-Viewer-Address
  • AWS managed: AllViewer, CORS-S3Origin, AllViewerExceptHostHeader

Invalidations — Evicting Cached Objects

🗑️ When to Invalidate
  • Deployed new version of a file with same filename
  • Emergency content removal (legal, security)
  • Cache-busting when TTL is very long
↓
✅ Better: Versioned filenames
  • app.v2.1.0.js instead of app.js
  • Old file cached, new file always fresh
  • No invalidation needed, no cost
⚠️ Invalidation pricing
  • First 1,000 paths/month: FREE
  • Additional: $0.005 per path
  • Wildcard /images/* counts as 1 path

Compression — Gzip & Brotli

📦 Automatic Compression

  • Enable "Compress objects automatically" on the behavior
  • CloudFront compresses text-based content (HTML, CSS, JS, JSON, XML) with gzip or Brotli
  • Brotli served if viewer sends Accept-Encoding: br — typically 15–25% smaller than gzip
  • Object must be 1 KB–10 MB to be compressed
  • Pre-compressed objects in S3: set Content-Encoding: gzip on the object — CloudFront won't re-compress
  • Compressed and uncompressed versions cached separately (Accept-Encoding is always in cache key)

CloudFront Function — URL Rewrite for SPA

CloudFront Function · JavaScript · SPA routing — serve index.html for all paths 
// Handler runs in under 1ms — no cold starts, scales to 10M req/s
function handler(event) {
    var request = event.request;
    var uri = request.uri;

    // If requesting a file with extension (e.g. .js, .css, .png) — serve as-is
    if (uri.match(/\.[a-zA-Z0-9]+$/)) {
        return request;
    }

    // All other paths (React/Vue/Angular routes) → rewrite to index.html
    request.uri = '/index.html';
    return request;
}

// ── Security headers function (Viewer Response trigger) ──
function handler(event) {
    var response = event.response;
    var headers  = response.headers;

    headers['strict-transport-security'] = { value: 'max-age=63072000; includeSubDomains; preload' };
    headers['x-content-type-options']      = { value: 'nosniff' };
    headers['x-frame-options']             = { value: 'DENY' };
    headers['referrer-policy']             = { value: 'strict-origin-when-cross-origin' };
    headers['permissions-policy']          = { value: 'camera=(), microphone=(), geolocation=()' };

    return response;
}

Lambda@Edge — JWT Validation (Node.js)

Lambda@Edge · Node.js · Viewer Request · JWT validation — protect private content 
const jwt = require('jsonwebtoken');

// Deployed to us-east-1, replicated globally by CloudFront
exports.handler = async (event) => {
    const request   = event.Records[0].cf.request;
    const headers   = request.headers;
    const authHeader = headers['authorization']?.[0]?.value;

    if (!authHeader?.startsWith('Bearer ')) {
        return unauthorised('Missing or invalid Authorization header');
    }

    try {
        const token   = authHeader.substring(7);
        const decoded = jwt.verify(token, process.env.JWT_PUBLIC_KEY, {
            algorithms: ['RS256']
        });

        // Inject claims as headers for origin to consume
        request.headers['x-user-id']    = [{ value: decoded.sub }];
        request.headers['x-user-email'] = [{ value: decoded.email }];

        return request;  // continue to origin
    } catch (err) {
        return unauthorised('Invalid or expired token');
    }
};

function unauthorised(message) {
    return {
        status: '401',
        statusDescription: 'Unauthorized',
        headers: { 'content-type': [{ value: 'application/json' }] },
        body: JSON.stringify({ error: message })
    };
}

🔐 Security — OAC, Signed URLs, WAF, Geo-restriction

OAC — Origin Access Control (Private S3)

OAC lets CloudFront access a private S3 bucket without making it public. CloudFront signs requests to S3 using SigV4 — S3 checks the signature and only allows requests from your specific distribution.

👤 User requests private file
↓
☁️ CloudFront Edge (cache miss)
  • OAC signs the S3 request with SigV4 using CloudFront's service principal
  • Adds Authorization, x-amz-date, x-amz-content-sha256 headers
↓
🗄️ Private S3 Bucket — Bucket Policy
  • Bucket policy: allow s3:GetObject only from cloudfront.amazonaws.com service principal
  • Condition: aws:SourceArn = your specific CloudFront distribution ARN
  • S3 verifies SigV4 signature → serves object to CloudFront
  • Direct S3 URL access returns 403 Forbidden ✅
S3 Bucket Policy — Allow only CloudFront OAC (not public) 
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid":       "AllowCloudFrontOAC",
    "Effect":   "Allow",
    "Principal": { "Service": "cloudfront.amazonaws.com" },
    "Action":   "s3:GetObject",
    "Resource": "arn:aws:s3:::my-private-bucket/*",
    "Condition": {
      "StringEquals": {
        // Only THIS distribution can access the bucket — not any CF distribution
        "aws:SourceArn": "arn:aws:cloudfront::123456789:distribution/E1ABCDEF123456"
      }
    }
  }]
}

Signed URLs & Signed Cookies

🔏 Signed URLs

  • Time-limited, access-controlled URL for a single file
  • Contains: expiry time, IP restriction (optional), signature
  • Best for: individual file downloads, video streaming links
  • Share via download button, email, API response
  • URL looks like: https://d1.cloudfront.net/video.mp4?Expires=1700000000&Signature=...&Key-Pair-Id=...

🍪 Signed Cookies

  • Time-limited access for multiple files (entire section)
  • 3 cookies: CloudFront-Expires, CloudFront-Signature, CloudFront-Key-Pair-Id
  • Best for: subscription video libraries, gated content sections, streaming playlists (HLS)
  • User logs in → set signed cookies → all subsequent requests to /premium/* work
C# · CloudFront Signed URL generator using RSA private key 
using System.Security.Cryptography;
using System.Text;

public static class CloudFrontSigner
{
    // Generate a time-limited CloudFront signed URL
    public static string CreateSignedUrl(
        string cloudFrontUrl,        // https://d1abc.cloudfront.net/video.mp4
        string keyPairId,            // CloudFront key pair ID (from console)
        string privateKeyPem,        // RSA private key in PEM format
        DateTime expiresUtc)         // when URL expires
    {
        var expireEpoch = (long)expiresUtc.Subtract(new DateTime(1970,1,1,0,0,0,DateTimeKind.Utc)).TotalSeconds();

        // Canned policy (simplest — just expiry + URL)
        var policy = $@"{{""Statement"":[{{""Resource"":""{cloudFrontUrl}"",""Condition"":{{""DateLessThan"":{{""AWS:EpochTime"":{expireEpoch}}}}}}}}]}}";
        var policyBytes = Encoding.UTF8.GetBytes(policy);

        // Sign with RSA private key (SHA-1 — CloudFront requirement)
        using var rsa = RSA.Create();
        rsa.ImportFromPem(privateKeyPem);
        var signature = rsa.SignData(policyBytes, HashAlgorithmName.SHA1, RSASignaturePadding.Pkcs1);

        // CloudFront-safe Base64 (replace +, =, /)
        var sig64 = Convert.ToBase64String(signature)
            .Replace('+', '-').Replace('=', '_').Replace('/', '~');
        var policy64 = Convert.ToBase64String(policyBytes)
            .Replace('+', '-').Replace('=', '_').Replace('/', '~');

        return $"{cloudFrontUrl}?Expires={expireEpoch}&Signature={sig64}&Key-Pair-Id={keyPairId}";
    }
}

// Usage — generate a signed URL valid for 1 hour
var signedUrl = CloudFrontSigner.CreateSignedUrl(
    "https://d1abc123.cloudfront.net/premium/video.mp4",
    "K2JCJMDEHXQW5F",      // Key pair ID from CloudFront console
    File.ReadAllText("cf-private-key.pem"),
    DateTime.UtcNow.AddHours(1));
// Return signedUrl to authenticated user only

AWS WAF Integration

🛡️ What WAF Adds to CloudFront

  • Block SQL injection and XSS attacks
  • Rate limiting by IP (e.g. 100 req/5min)
  • Managed rules: OWASP Top 10, AWS managed threat lists
  • Geo-blocking at WAF level (more granular than CF geo-restriction)
  • Block/allow specific IPs or IP ranges
  • Custom rules: block User-Agents, body content, URI patterns
  • WAF logs to CloudWatch, S3, or Kinesis Firehose

🌍 Geo-restriction

  • Built into CloudFront — no WAF needed for basic geo blocking
  • Allow list: only serve to listed countries
  • Block list: deny requests from listed countries
  • Uses CloudFront's own IP geolocation database
  • Blocked users receive 403 Forbidden
  • Custom error page: configure 403 → your /geo-blocked.html
  • Lambda@Edge for more granular logic (city, ISP, etc.)

⚡ Performance — Optimising CloudFront

Cache Hit Ratio Optimisation

  • Use versioned filenames (app.a1b2c3.js) instead of invalidations — highest-ROI change for static assets
  • Set long TTLs (1 year = 31536000) on immutable assets — versioning handles freshness
  • Minimise cache key components — only include headers/cookies that actually change the response
  • Normalise query strings in CloudFront Function before they hit the cache key
  • Separate static and dynamic behaviors — long TTL for static, 0 TTL for dynamic
  • Use Cache-Control: public, max-age=31536000, immutable for versioned assets from origin

Origin Shield — Extra Cache Layer

🌍 Edge PoPs (600+)
  • First cache tier
  • Cache miss → Origin Shield
🛡️ Origin Shield
  • Optional regional cache layer
  • Consolidates all cache misses from all PoPs
  • ~$0.0025–0.0150/10k requests
🏠 Origin
  • Receives far fewer requests
  • Lower load, lower cost

💡 When to Enable Origin Shield

Enable Origin Shield when your origin is expensive to call (on-prem, slow DB, rate-limited third-party API) or when your content is globally distributed. The overhead of Origin Shield requests (<$15/month for most) is far less than origin compute savings. Choose the Origin Shield region closest to your origin.

Pattern 1 — Create Invalidation (Flush Cache)

C# · CloudFront SDK · Create cache invalidation 
using Amazon.CloudFront;
using Amazon.CloudFront.Model;

public class CloudFrontManager
{
    private readonly IAmazonCloudFront _cf;

    public CloudFrontManager(IAmazonCloudFront cf) => _cf = cf;

    // Invalidate specific paths after deployment
    public async Task<string> InvalidatePathsAsync(
        string distributionId,
        IList<string> paths)   // e.g. ["/index.html", "/static/*"]
    {
        var request = new CreateInvalidationRequest
        {
            DistributionId = distributionId,
            InvalidationBatch = new InvalidationBatch
            {
                // CallerReference must be unique per invalidation request
                CallerReference = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds().ToString(),
                Paths = new Paths
                {
                    Quantity = paths.Count,
                    Items    = paths.ToList()
                }
            }
        };

        var response = await _cf.CreateInvalidationAsync(request);
        // Returns invalidation ID — use GetInvalidation to poll status
        return response.Invalidation.Id;
    }

    // Invalidate everything — use after full redeployment
    public Task<string> InvalidateAllAsync(string distributionId)
        => InvalidatePathsAsync(distributionId, new[] { "/*" });

    // Wait for invalidation to complete (~30–60 seconds)
    public async Task WaitForInvalidationAsync(string distributionId, string invalidationId)
    {
        while (true)
        {
            var resp = await _cf.GetInvalidationAsync(new GetInvalidationRequest
            {
                DistributionId = distributionId,
                Id             = invalidationId
            });

            if (resp.Invalidation.Status == "Completed") break;
            await Task.Delay(TimeSpan.FromSeconds(5));
        }
    }
}

    Console.WriteLine($"Status:  {dist.Status}");                     // Deployed / InProgress
    Console.WriteLine($"Enabled: {dist.DistributionConfig.Enabled}");
    Console.WriteLine($"Origins: {dist.DistributionConfig.Origins.Quantity}");
    Console.WriteLine($"Aliases: {string.Join(", ", dist.DistributionConfig.Aliases.Items)}");
    return dist;

// List all distributions in account
public async Task<List<DistributionSummary>> ListDistributionsAsync()
{
    var distributions = new List<DistributionSummary>();
    string? marker = null;

    do
    {
        var resp = await _cf.ListDistributionsAsync(new ListDistributionsRequest
        {
            Marker  = marker,
            MaxItems = "100"
        });

        distributions.AddRange(resp.DistributionList.Items);
        marker = resp.DistributionList.IsTruncated ? resp.DistributionList.NextMarker : null;
    }
    while (marker != null);

    return distributions;
}

// Enable or disable a distribution (e.g. for emergency maintenance)
public async Task SetDistributionEnabledAsync(string distributionId, bool enabled)
{
    var current = await _cf.GetDistributionConfigAsync(new GetDistributionConfigRequest
        { Id = distributionId });

    current.DistributionConfig.Enabled = enabled;

    await _cf.UpdateDistributionAsync(new UpdateDistributionRequest
    {
        Id                 = distributionId,
        DistributionConfig = current.DistributionConfig,
        IfMatch            = current.ETag   // optimistic locking — required by CloudFront API
    });
}

Pattern 3 — CI/CD Deploy Pipeline (S3 + CloudFront Invalidation)

C# · Deploy script · Upload to S3 · Invalidate CloudFront 
using Amazon.S3;
using Amazon.S3.Transfer;
using Amazon.CloudFront;

public class DeploymentService
{
    private readonly IAmazonS3         _s3;
    private readonly IAmazonCloudFront _cf;
    private readonly CloudFrontManager  _cfm;

    // Full deploy: upload build folder to S3 then invalidate CF cache
    public async Task DeployAsync(
        string buildFolder,       // local path: ./dist
        string s3Bucket,          // my-react-app-prod
        string distributionId)    // CloudFront distribution ID
    {
        Console.WriteLine("📦 Uploading build to S3...");

        var transfer = new TransferUtility(_s3);
        await transfer.UploadDirectoryAsync(new TransferUtilityUploadDirectoryRequest
        {
            Directory      = buildFolder,
            BucketName     = s3Bucket,
            SearchOption   = SearchOption.AllDirectories,
            CannedACL      = S3CannedACL.Private,   // private — served via CloudFront only
            StorageClass   = S3StorageClass.Standard,
            // Set long cache headers on versioned assets
        });

        Console.WriteLine("🗑️ Invalidating CloudFront cache...");
        var invalidationId = await _cfm.InvalidatePathsAsync(
            distributionId,
            new[] { "/index.html", "/asset-manifest.json" });
        // Only invalidate entry points — versioned assets auto-bust via filename hash

        Console.WriteLine("⏳ Waiting for invalidation to complete...");
        await _cfm.WaitForInvalidationAsync(distributionId, invalidationId);

        Console.WriteLine("✅ Deployment complete!");
    }
}
