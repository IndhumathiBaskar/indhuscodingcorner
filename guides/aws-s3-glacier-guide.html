Ultra-low-cost cloud archive storage for data that is rarely accessed — built for compliance, backup, and long-term retention at cents per GB per month.

🏗️ What is S3 Glacier?

Three archive storage classes inside Amazon S3 — designed for data you keep for years but rarely need to access.

One-liner S3 Glacier = archive storage at $0.004–$0.00099/GB/month (vs S3 Standard $0.023/GB). Trade retrieval speed for massive cost savings. Three tiers: Instant, Flexible, and Deep Archive.
🧊 Glacier Instant Retrieval

Millisecond access like S3 Standard — but 68% cheaper for storage. Access once per quarter. Same retrieval latency as S3.

$0.004/GB/month

⏱️ Glacier Flexible Retrieval

Formerly "S3 Glacier". Minutes to hours retrieval. Three speeds: Expedited (1–5 min), Standard (3–5 hr), Bulk (5–12 hr).

$0.0036/GB/month

🏔️ Glacier Deep Archive

Lowest cost storage on AWS. 7–12 hour retrieval. For data accessed once or twice a year. Ideal for regulatory archives kept 7–10 years.

$0.00099/GB/month

Key Concepts Archive — a single object stored in Glacier (up to 40 TB). Vault — container for archives (like an S3 bucket, legacy Glacier API only). S3 Storage Class — modern way: store in S3 bucket, set storage class to Glacier. Vault Lock — immutable compliance policy (WORM).

Two Ways to Use Glacier

✅ Via S3 (Recommended)

Upload to S3 bucket, set object storage class to GLACIER or DEEP_ARCHIVE. Use S3 Lifecycle rules to automatically transition objects. Use S3 API (RestoreObject) for retrieval. Full S3 feature set — versioning, replication, inventory.

⚠️ Via Glacier API (Legacy)

Direct Glacier API with Vaults and Archives. Separate namespace from S3. Requires job-based retrieval (async). Used for legacy integrations. New workloads should use S3 + Glacier storage class.

S3 RestoreObject — How It Works

Glacier objects in S3 are not directly readable You must first initiate a restore job via RestoreObject. AWS creates a temporary copy in S3 Standard for the duration you specify (days). After that window expires, the temporary copy is deleted — the original Glacier object remains.
  • 1
    Initiate Restore (RestoreObject)Call RestoreObject with retrieval tier and number of days to keep the restored copy available.
  • 2
    Wait for RestorePoll HeadObject for x-amz-restore header. Or set up S3 Event Notification → SNS/SQS/Lambda on s3:ObjectRestore:Completed.
  • 3
    Download Restored CopyOnce restored, use GetObject as normal. The temporary copy is available in S3 Standard for the specified number of days.
  • 4
    Restore ExpiresAfter the restore window closes, the temporary copy is deleted. The original archive in Glacier remains intact and can be restored again.
RESTORE FLOW (S3 + Glacier Flexible):

App → RestoreObject(bucket, key, days=7, tier=Standard)
  └── Glacier job created → queued

Wait 3–5 hours (Standard tier)

Glacier job completes →
  ├── Temp copy created in S3 Standard
  └── s3:ObjectRestore:Completed event → SNS → Lambda

App → GetObject(bucket, key) ← downloads temp copy

After 7 days → temp copy deleted (original Glacier archive intact)

📋 S3 Lifecycle Policies

Automatically transition objects between storage classes and expire them — set it once, save forever.

Lifecycle Rules = Automatic Cost Optimisation Define rules on an S3 bucket: after N days, move objects to a cheaper class. After M days, move to Glacier. After Y days, delete. Objects transition automatically — no code needed.

Common Lifecycle Pattern — 3-Tier Archive

JSON S3 Lifecycle configuration — transition to Glacier then Deep Archive 
{
  "Rules": [
    {
      "ID": "ArchiveAndExpire",
      "Status": "Enabled",
      "Filter": { "Prefix": "logs/" },
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER_IR"    // Glacier Instant Retrieval
        },
        {
          "Days": 180,
          "StorageClass": "GLACIER"        // Glacier Flexible Retrieval
        },
        {
          "Days": 365,
          "StorageClass": "DEEP_ARCHIVE"   // Glacier Deep Archive
        }
      ],
      "Expiration": {
        "Days": 2555   // Delete after 7 years
      },
      "NoncurrentVersionTransitions": [
        {
          "NoncurrentDays": 30,
          "StorageClass": "GLACIER"
        }
      ],
      "NoncurrentVersionExpiration": {
        "NoncurrentDays": 90
      }
    }
  ]
}
🎯 Lifecycle Filter Options

Prefix — apply to objects with key prefix (e.g. logs/)

Tag — apply to objects with specific tags (e.g. archive=true)

Object size — min/max size filter (e.g. only objects > 128 KB)

No filter — apply to entire bucket

⚙️ Lifecycle Actions

Transition — move to cheaper storage class after N days

Expiration — permanently delete object after N days

NoncurrentVersion — manage old versions (versioned bucket)

AbortIncompleteMultipartUpload — clean up stuck uploads

🔒 Vault Lock & Object Lock

Enforce immutability — prevent deletion or modification of archived data for compliance (WORM).

🔒 S3 Glacier Vault Lock (Legacy API)

Attach a Vault Lock policy to a Glacier vault. Policy is locked — cannot be changed or deleted once confirmed. Enforces WORM (Write Once Read Many) compliance.

Supports conditions like minimum retention period, MFA delete, and deny all deletes.

Legacy Glacier API

🔐 S3 Object Lock (Modern)

Per-object immutability on S3 buckets with versioning enabled. Two modes: Governance (admin can override) and Compliance (no one can delete — not even root).

Works with Glacier storage classes. The recommended approach for new workloads.

Modern S3 API

S3 Object Lock Modes

ModeWho Can Delete?Use Case
Governance Mode Users with s3:BypassGovernanceRetention permission Internal compliance with admin override ability
Compliance Mode Nobody — not even AWS root account SEC 17a-4, FINRA, HIPAA — strict regulatory WORM
Legal Hold Nobody while hold active — no expiry date Litigation holds — turn on/off without retention date

Vault Lock Policy Example

JSON Glacier Vault Lock — deny all deletes (WORM) 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyDeletion",
      "Effect": "Deny",
      "Principal": "*",
      "Action": [
        "glacier:DeleteArchive",
        "glacier:DeleteVault"
      ],
      "Resource": "*"
    },
    {
      "Sid": "EnforceMinRetention",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "glacier:DeleteArchive",
      "Resource": "*",
      "Condition": {
        "NumericLessThan": {
          "glacier:ArchiveAgeInDays": "365"
        }
      }
    }
  ]
}
⚠️ Vault Lock is PERMANENT Once a Vault Lock policy is completed (confirmed), it cannot be changed or deleted. You have a 24-hour window after initiating to abort the lock. After confirmation — it's permanent, even for the AWS account root user.

🧩 C# / .NET SDK Examples

Using AWSSDK.S3 for S3 + Glacier storage classes (recommended approach).

NuGet dotnet add package AWSSDK.S3 — use S3 API with Glacier storage classes. For legacy Glacier API: AWSSDK.Glacier.
C# Upload directly to Glacier Instant Retrieval 
using Amazon.S3;
using Amazon.S3.Model;

var s3 = new AmazonS3Client();

// Upload with Glacier Instant Retrieval storage class
await s3.PutObjectAsync(new PutObjectRequest
{
    BucketName   = "my-archive-bucket",
    Key          = "backups/2024/annual-report.pdf",
    FilePath     = "C:\\Reports\\annual-report.pdf",
    StorageClass = S3StorageClass.GlacierInstantRetrieval,  // GLACIER_IR
    ServerSideEncryptionMethod = ServerSideEncryptionMethod.AWSKMS,
    ServerSideEncryptionKeyManagementServiceKeyId = "arn:aws:kms:..."
});

// Other storage class options:
// S3StorageClass.Glacier          → Glacier Flexible Retrieval
// S3StorageClass.DeepArchive      → Glacier Deep Archive
// S3StorageClass.IntelligentTiering
C# Initiate restore from Glacier Flexible Retrieval 
// Step 1: Initiate restore job
await s3.RestoreObjectAsync(new RestoreObjectRequest
{
    BucketName = "my-archive-bucket",
    Key        = "backups/2024/annual-report.pdf",
    Days       = 7,   // keep temp copy available for 7 days
    RestoreRequestType = RestoreRequestType.SELECT,
    RetrievalTier      = GlacierJobTier.Standard  // Expedited | Standard | Bulk
});

// Step 2: Poll until restored
while (true)
{
    var meta = await s3.GetObjectMetadataAsync(new GetObjectMetadataRequest
        { BucketName = "my-archive-bucket", Key = "backups/2024/annual-report.pdf" });

    var restoreHeader = meta.RestoreInProgress;

    if (!restoreHeader)
    {
        Console.WriteLine("✅ Restore complete — object is now readable");
        break;
    }

    Console.WriteLine("⏳ Restore in progress...");
    await Task.Delay(TimeSpan.FromMinutes(5));
}

// Step 3: Download restored object
var obj = await s3.GetObjectAsync("my-archive-bucket", "backups/2024/annual-report.pdf");
using var fs = File.Create("restored-report.pdf");
await obj.ResponseStream.CopyToAsync(fs);
C# Set S3 Lifecycle rules via SDK 
using Amazon.S3.Model;

await s3.PutLifecycleConfigurationAsync(new PutLifecycleConfigurationRequest
{
    BucketName = "my-archive-bucket",
    Configuration = new LifecycleConfiguration
    {
        Rules =
        [
            new LifecycleRule
            {
                Id     = "ArchiveLogs",
                Status = LifecycleRuleStatus.Enabled,
                Filter = new LifecycleFilter
                {
                    LifecycleFilterPredicate = new LifecyclePrefixPredicate
                        { Prefix = "logs/" }
                },
                Transitions =
                [
                    new LifecycleTransition
                        { Days = 90,  StorageClassTransition = S3StorageClass.GlacierInstantRetrieval },
                    new LifecycleTransition
                        { Days = 365, StorageClassTransition = S3StorageClass.DeepArchive }
                ],
                Expiration = new LifecycleRuleExpiration { Days = 2555 }
            }
        ]
    }
});
C# Enable S3 Object Lock on upload (WORM) 
// Bucket must have Object Lock enabled at creation
await s3.PutObjectAsync(new PutObjectRequest
{
    BucketName   = "my-compliant-archive",
    Key          = "financial/2024/q4-report.pdf",
    FilePath     = "q4-report.pdf",
    StorageClass = S3StorageClass.DeepArchive,
    ObjectLockMode            = ObjectLockMode.Compliance,
    ObjectLockRetainUntilDate = DateTime.UtcNow.AddYears(7)  // lock for 7 years
});

// Add Legal Hold (no expiry date)
await s3.PutObjectLegalHoldAsync(new PutObjectLegalHoldRequest
{
    BucketName = "my-compliant-archive",
    Key        = "financial/2024/q4-report.pdf",
    LegalHold  = new ObjectLockLegalHold
        { Status = ObjectLockLegalHoldStatus.On }
});

💡 Best Practices

Get the most out of Glacier — cost, compliance, and retrieval efficiency.

✅ Use Lifecycle Rules

Never manually move objects to Glacier. Define lifecycle rules at bucket or prefix level — automatic, consistent, zero operational overhead.

✅ Choose the Right Tier

Access quarterly? → Glacier Instant. Access 1–2× per year, tolerate hours? → Glacier Flexible. Access once per year, tolerate days? → Deep Archive.

✅ Avoid Early Deletion

Don't delete Glacier objects before minimum duration (90/180 days). You'll be charged for the full minimum. Use lifecycle expiration to manage deletions automatically after the minimum.

✅ Use S3 Event Notifications

Don't poll for restore completion. Set up s3:ObjectRestore:Completed event → SNS or Lambda. Notification arrives when restore is ready — no wasted API calls.

✅ Tag Archives for Governance

Tag objects with retention-years, compliance-type, data-owner. Use tag-based lifecycle rules and cost allocation tags to track archive costs by project.

✅ Enable S3 Inventory

Enable S3 Inventory to get a daily/weekly CSV of all objects + their storage class, size, encryption status. Query with Athena for audit reports.

✅ Encrypt Everything

Enable SSE-S3 or SSE-KMS on the bucket. For compliance data, use CMK (customer-managed key) so you control key rotation and can audit key usage in CloudTrail.

✅ Test Restores Regularly

Schedule quarterly restore drills. Verify that archived data is actually retrievable before you need it in a real DR or compliance audit scenario. Document restore time per tier.
