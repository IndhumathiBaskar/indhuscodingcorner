📦 What is AWS CodeCommit?

AWS CodeCommit is a fully managed, private Git repository service hosted in AWS. It works exactly like GitHub or GitLab — you use the same git clone, git push, git pull commands — but your code stays within your AWS account, behind IAM, never leaving your security boundary.

💡 The Core Value Proposition

CodeCommit is Git — no new skills needed. The difference is that authentication is via IAM credentials instead of GitHub tokens, your code never leaves AWS, it natively integrates with CodePipeline / CodeBuild / CodeDeploy, and there are no per-seat fees — you pay only for storage and API calls.

⚠️ Important — CodeCommit is Being Deprecated

AWS announced in July 2024 that CodeCommit is no longer accepting new customers. Existing customers can continue using it, but AWS recommends migrating to GitHub, GitLab, or Bitbucket. This guide covers CodeCommit fully for existing users and migration paths for those planning to move.

Branch Commands

bash · Branch Management 
# Create and switch to a new branch
git checkout -b feature/WAT-123-add-payments
# or (modern Git)
git switch -c feature/WAT-123-add-payments

# Push branch to CodeCommit
git push -u origin feature/WAT-123-add-payments

# Protect main branch (via AWS CLI) ────────────────────────────
# CodeCommit uses approval rules on PRs instead of branch protection
# Create an approval rule template that requires 2 approvals on main
aws codecommit create-approval-rule-template \
  --approval-rule-template-name RequireTwoApprovals \
  --approval-rule-template-content '{
    "Version": "2018-11-08",
    "DestinationReferences": ["refs/heads/main"],
    "Statements": [{
      "Type": "Approvers",
      "NumberOfApprovalsNeeded": 2,
      "ApprovalPoolMembers": [
        "arn:aws:iam::123456789012:root"
      ]
    }]
  }'

# Associate the template with your repo
aws codecommit associate-approval-rule-template-with-repository \
  --repository-name watson-api \
  --approval-rule-template-name RequireTwoApprovals

# Delete a branch (local + remote)
git branch -d feature/WAT-123-add-payments
git push origin --delete feature/WAT-123-add-payments

💡 Trunk-Based Development Alternative

For smaller teams or microservices, use trunk-based development — everyone commits directly to main (or short-lived feature branches <2 days). Combined with feature flags, this keeps the branch count low and reduces merge conflicts. Great for Lambda microservices where each repo has one function.

Best Practices & Common Mistakes

  • Use git-remote-codecommit with AWS SSO — no static credentials to rotate or accidentally commit
  • Protect main and develop branches with Approval Rule Templates — require at least 2 approvals before merging
  • Deny direct pushes to main/develop via IAM Condition on codecommit:References — enforce PR-only workflow
  • Set up Notification Rules (not just Triggers) for PR events — they carry richer metadata and work with Chatbot/Slack
  • Tag every repository with Team, CostCenter, and Owner tags — CodeCommit repos appear in Cost Explorer
  • Use DetectionMode: EVENTS in CodePipeline source — faster than polling, uses EventBridge
  • Keep buildspec.yml in the root of the repository — versioned alongside the code it builds
  • Plan your migration to GitHub/GitLab — CodeCommit is closed to new customers as of July 2024

⚠️ Top CodeCommit Mistakes

❌ Storing HTTPS Git credentials in code or CI/CD env vars — use IAM roles or git-remote-codecommit instead
❌ No approval rules on main — anyone with push access can directly commit to production branch
❌ Using polling in CodePipeline — polling is slow (1 min delay) and costs money; use EventBridge detection
❌ Not tagging repositories — impossible to attribute costs or find repos by team later
❌ Committing large binary files — use S3 or Git LFS alternatives; large files slow clone times for the entire team
❌ Single IAM user for all CI/CD — use separate IAM roles per pipeline with least-privilege access
❌ Not planning migration — CodeCommit is closed to new customers; existing teams should migrate to GitHub/GitLab
