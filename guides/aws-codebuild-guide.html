🔨 What is AWS CodeBuild?

AWS CodeBuild is a fully managed continuous integration (CI) service that compiles source code, runs unit tests, and produces deployment-ready artifacts. You pay only per build minute — no servers, no agents, no Jenkins to maintain. CodeBuild spins up a fresh container for each build, runs your buildspec.yml, and shuts down.

💡 The Core Mental Model

Think of CodeBuild as a clean room on demand. Every build starts from a pristine container image (Amazon Linux, Ubuntu, Windows, or your own Docker image), runs your commands in order, uploads results to S3, and disappears. No state bleeds between builds — unless you configure caching.

buildspec.yml

YAML file defining phases, env vars, cache, artifacts. Lives in your repo root.

The CodeBuild configuration — source, environment, service role, artifacts, cache.

One execution of a project. Has its own logs, status, and artifact output.

The container image + compute type used. Managed images or your own Docker image.

💾

Speed up builds by caching dependencies (NuGet, npm) in S3 between runs.

Artifacts

Output files uploaded to S3. Consumed by next pipeline stage (CodeDeploy, CFN).

VPC Support

Run builds inside your VPC to access RDS, ElastiCache, or private endpoints.

version: 0.2

env:
  # Plain environment variables
  variables:
    DOTNET_CLI_TELEMETRY_OPTOUT: "1"
    ARTIFACT_BUCKET:           "watson-artifacts-eu-west-1"
    ENVIRONMENT:               "dev"

  # Secrets from SSM Parameter Store (value NOT shown in logs)
  parameter-store:
    SONAR_TOKEN: "/watson/ci/sonar-token"
    SLACK_HOOK:  "/watson/ci/slack-webhook"

  # Secrets from Secrets Manager
  secrets-manager:
    NUGET_API_KEY: "watson/nuget-api-key:apiKey"

  # Export variables to be available in later phases
  exported-variables:
    - IMAGE_TAG
    - BUILD_VERSION

phases:
  install:
    runtime-versions:
      dotnet: 8
    commands:
      - pip install aws-sam-cli
      - sam --version
      - dotnet tool install --global dotnet-coverage

  pre_build:
    commands:
      - echo "Pre-build started at $(date)"
      # Set build version from git tag or commit short SHA
      - export BUILD_VERSION=$(git describe --tags --always --dirty)
      - export IMAGE_TAG=${CODEBUILD_BUILD_NUMBER}
      # Login to ECR for Docker base images
      - aws ecr get-login-password --region eu-west-1 | docker login --username AWS --password-stdin 123456789012.dkr.ecr.eu-west-1.amazonaws.com
      # Restore NuGet packages (uses cache)
      - dotnet restore ./src/WatsonOrders/WatsonOrders.csproj

  build:
    on-failure: ABORT   # Stop immediately on build failure
    commands:
      - echo "Building .NET Lambda..."
      - dotnet build ./src/WatsonOrders/ --configuration Release --no-restore

      # Run unit tests with coverage
      - dotnet-coverage collect "dotnet test ./tests/WatsonOrders.Tests/ --no-build --logger trx --results-directory /tmp/test-results" --output /tmp/coverage.xml --output-format xml

      # SAM build (compiles + packages Lambda)
      - sam build --cached --parallel

      # SAM package — upload to S3, generate packaged template
      - sam package --output-template-file packaged.yaml --s3-bucket $ARTIFACT_BUCKET --s3-prefix watson-api/$BUILD_VERSION

  post_build:
    commands:
      - echo "Build completed: $BUILD_VERSION"
      # Notify Slack on success
      - |
          if [ "$CODEBUILD_BUILD_SUCCEEDING" = "1" ]; then
            curl -s -X POST -H 'Content-type: application/json' \
              --data "{\"text\":\"✅ Watson build $BUILD_VERSION succeeded\"}" \
              $SLACK_HOOK
          fi

reports:
  WatsonTestResults:
    files:
      - "**/*.trx"
    base-directory:  /tmp/test-results
    file-format:     VisualStudioTrx
  WatsonCoverage:
    files:
      - "/tmp/coverage.xml"
    file-format:     CloverXml

artifacts:
  files:
    - packaged.yaml
    - samconfig.toml
  name:             watson-build-$BUILD_VERSION
  discard-paths:    yes

cache:
  paths:
    - "/root/.nuget/packages/**/*"
    - "/root/.dotnet/**/*"
    - ".aws-sam/cache/**/*"

💾 Caching — Speed Up Every Build

Without caching, CodeBuild downloads NuGet packages, npm modules, and pip packages on every build — adding 30–90 seconds. With S3 caching, subsequent builds restore the cache in ~5 seconds.

Cache Types

☁️ S3 Cache

Cache stored in S3. Shared across all builds regardless of which compute instance runs. Best for teams with concurrent builds. Slight overhead (~5s) to download/upload cache.

💻 Local Cache

Cache kept on the build host between builds. Faster restore (~1s) but only works if the same host runs your next build — not guaranteed. Three modes: SOURCE, DOCKER_LAYER, CUSTOM.

🔀 Combined

Use both S3 + local. Local cache used when available (same host), S3 cache used as fallback. Best of both worlds for high-frequency build projects.

Cache Configuration — buildspec.yml

YAML · Cache Configuration for .NET + SAM 
# In buildspec.yml
cache:
  paths:
    # NuGet package cache
    - "/root/.nuget/packages/**/*"
    # .NET SDK + tools
    - "/root/.dotnet/**/*"
    # SAM build cache
    - ".aws-sam/cache/**/*"
    # npm packages (if you have a frontend)
    - "node_modules/**/*"
    # pip packages
    - "/root/.cache/pip/**/*"

# In CodeBuild project definition (CloudFormation)
Cache:
  Type:     S3
  Location: watson-artifacts-eu-west-1/codebuild-cache

# Or local cache with Docker layer support (for Docker builds)
Cache:
  Type:  LOCAL
  Modes:
    - LOCAL_SOURCE_CACHE    # Caches the source code download
    - LOCAL_DOCKER_LAYER_CACHE # Caches Docker image layers
    - LOCAL_CUSTOM_CACHE    # Uses paths from buildspec cache section

💡 Cache Invalidation Strategy

Cache is keyed by project + branch. To force a full refresh, either delete the cache from S3 manually, or temporarily rename the cache path in buildspec.yml. The cache updates automatically at the end of each successful build — so a cache miss on build #1 will be a cache hit on build #2.

Test + Coverage Setup — C# .NET

YAML · buildspec.yml — .NET Test with TRX + Coverage 
phases:
  install:
    commands:
      - dotnet tool install --global dotnet-coverage
      - dotnet tool install --global dotnet-reportgenerator-globaltool

  build:
    commands:
      # Run tests and generate TRX + coverage in one command
      - |
          dotnet-coverage collect \
            "dotnet test ./tests/ \
              --logger trx \
              --results-directory /tmp/test-results \
              --blame-hang-timeout 60s \
              --no-build" \
            --output /tmp/coverage.xml \
            --output-format xml

      # Generate HTML coverage report
      - reportgenerator -reports:/tmp/coverage.xml -targetdir:/tmp/coverage-report -reporttypes:Html

      # Fail build if coverage < 80%
      - |
          COVERAGE=$(grep -oP 'line-rate="\K[^"]+' /tmp/coverage.xml | head -1)
          echo "Coverage: $COVERAGE"
          if (( $(echo "$COVERAGE < 0.80" | bc -l) )); then
            echo "❌ Coverage $COVERAGE is below 80% threshold"
            exit 1
          fi

  TestResults:
    files:
      - "**/*.trx"
    base-directory: /tmp/test-results
    file-format:   VisualStudioTrx
  CodeCoverage:
    files:

CodeBuild Automation Service

C# · CodeBuildService.cs — Start, Poll, List, Report 
using Amazon.CodeBuild;
using Amazon.CodeBuild.Model;

public class CodeBuildService
{
    private readonly IAmazonCodeBuild _cb;

    public CodeBuildService(IAmazonCodeBuild cb) => _cb = cb;

    // ── START A BUILD ─────────────────────────────────────
    public async Task<string> StartBuildAsync(
        string projectName,
        string? sourceVersion = null,
        Dictionary<string, string>? envOverrides = null,
        CancellationToken ct = default)
    {
        var request = new StartBuildRequest
        {
            ProjectName   = projectName,
            SourceVersion = sourceVersion   // branch, tag, or commit SHA
        };

        if (envOverrides?.Count > 0)
        {
            request.EnvironmentVariablesOverride = envOverrides
                .Select(kv => new EnvironmentVariable
                {
                    Name  = kv.Key,
                    Value = kv.Value,
                    Type  = EnvironmentVariableType.PLAINTEXT
                }).ToList();
        }

        var resp = await _cb.StartBuildAsync(request, ct);
        return resp.Build.Id;
    }

    // ── WAIT FOR BUILD TO COMPLETE ────────────────────────
    public async Task<Build> WaitForBuildAsync(
        string buildId,
        TimeSpan? pollInterval = null,
        CancellationToken ct = default)
    {
        var interval = pollInterval ?? TimeSpan.FromSeconds(15);

        while (!ct.IsCancellationRequested)
        {
            var resp = await _cb.BatchGetBuildsAsync(
                new BatchGetBuildsRequest { Ids = [buildId] }, ct);

            var build = resp.Builds.First();

            Console.WriteLine($"[{DateTime.UtcNow:T}] Build {buildId}: {build.BuildStatus} — Phase: {build.CurrentPhase}");

            if (build.BuildComplete)
                return build;

            await Task.Delay(interval, ct);
        }

        throw new OperationCanceledException("Timed out waiting for build.");
    }

    // ── GET BUILD PHASES ──────────────────────────────────
    public async Task<List<BuildPhase>> GetPhasesAsync(
        string buildId,
        CancellationToken ct = default)
    {
        var resp = await _cb.BatchGetBuildsAsync(
            new BatchGetBuildsRequest { Ids = [buildId] }, ct);

        return resp.Builds.First().Phases;
    }

    // ── LIST BUILDS FOR PROJECT ───────────────────────────
    public async Task<List<string>> ListBuildsAsync(
        string projectName,
        int maxResults = 10,
        CancellationToken ct = default)
    {
        var resp = await _cb.ListBuildsForProjectAsync(
            new ListBuildsForProjectRequest
            {
                ProjectName = projectName,
                SortOrder   = SortOrderType.DESCENDING
            }, ct);

        return resp.Ids.Take(maxResults).ToList();
    }

    // ── STOP A RUNNING BUILD ──────────────────────────────
    public async Task StopBuildAsync(
        string buildId,
        CancellationToken ct = default)
        => await _cb.StopBuildAsync(
            new StopBuildRequest { Id = buildId }, ct);

    // ── FULL BUILD + WAIT CONVENIENCE METHOD ──────────────
    public async Task<Build> BuildAndWaitAsync(
        string projectName,
        string? branch = null,
        Dictionary<string, string>? env = null,
        CancellationToken ct = default)
    {
        var buildId = await StartBuildAsync(projectName, branch, env, ct);
        Console.WriteLine($"Started build: {buildId}");

        var result = await WaitForBuildAsync(buildId, ct: ct);

        if (result.BuildStatus != "SUCCEEDED")
        {
            // Log failed phases
            var failedPhases = result.Phases
                .Where(p => p.PhaseStatus == "FAILED")
                .Select(p => $"{p.PhaseType}: {string.Join(", ", p.Contexts.Select(c => c.Message))}");

            throw new Exception(
                $"Build {buildId} {result.BuildStatus}.\nFailed phases:\n{string.Join("\n", failedPhases)}");
        }

        return result;
    }
}

Best Practices & Common Mistakes

  • Always use runtime-versions in the install phase — specifying dotnet: 8 ensures the correct SDK version, regardless of which image version is used
  • Use SSM Parameter Store or Secrets Manager for secrets in env.parameter-store or env.secrets-manager — never hardcode secrets in buildspec.yml or project config
  • Enable S3 caching for NuGet packages — reduces build time by 40–60% on typical .NET projects
  • Set on-failure: ABORT in the build phase — stops wasting build minutes when compilation fails
  • Always publish test results via the reports section — gives you trend visibility across builds
  • Use CODEBUILD_BUILD_SUCCEEDING in post_build to conditionally run steps (e.g., only notify Slack on success)
  • Grant CodeBuild's IAM role only the permissions it needs — S3 for artifacts, ECR for images, SSM for parameters
  • Use BUILD_GENERAL1_MEDIUM (4 vCPU / 7 GB) for standard .NET builds — SMALL is often CPU-bottlenecked on compilation

⚠️ Top CodeBuild Mistakes

❌ Hardcoding secrets in buildspec.yml — visible to anyone with repo access; use SSM/Secrets Manager
❌ No cache configured — downloading all NuGet packages every build wastes minutes and money
❌ Privileged mode always on — only enable when building Docker images; it's a security risk otherwise
❌ Using General1_LARGE for simple builds — 4x cost of MEDIUM; only use for Docker builds or massive test suites
❌ No timeout configured — default is 60 min; a hung build costs money; set a realistic timeout (e.g., 10 min)
❌ Not publishing test reports — failing builds with no visible test output make debugging slow
❌ Running integration tests in unit test phase — slow tests in every PR build; separate fast unit tests from slower integration tests using batch builds
