🌱 BEGINNER LEVEL — Cloud Foundations

☁️ What is Cloud Computing?

On-demand delivery of IT resources over the internet with pay-as-you-go pricing.

🏢 Traditional vs Cloud

Traditional: Buy servers → wait weeks → over-provision → pay for idle.

Cloud: Launch in seconds → scale instantly → pay only for what you use.

📦 Service Models

IaaS EC2, S3 — you manage OS up

PaaS Elastic Beanstalk, RDS — AWS manages runtime

SaaS WorkMail — fully managed app

🌍 Global Infrastructure

Regions — 33+ geographic locations (us-east-1, eu-west-1…)

AZs — 2–6 isolated data centres per region

Edge Locations — 400+ for CloudFront CDN

🖥️ EC2 — Elastic Compute Cloud

Virtual machines in the cloud. The foundational compute service.

📋 EC2 Instance Types

t3/t4g — General purpose, burstable. Web servers, dev.

m6i/m7i — Balanced CPU/RAM. App servers.

c6i/c7i — Compute optimised. CPU-heavy workloads.

r6i/r7i — Memory optimised. In-memory DBs, caches.

p4/g5 — GPU. ML training, video rendering.

💰 EC2 Pricing Models

On-Demand Pay per hour/second. No commitment. Most expensive.

Reserved 1–3 year commitment. Up to 72% discount.

Savings Plans Commit to $/hr spend. 66% discount.

Spot Unused capacity. Up to 90% discount. Can be interrupted.

Dedicated Host Physical server. Compliance/licensing.

🪣 S3 — Simple Storage Service

📦 S3 Key Concepts

Bucket — globally unique container for objects

Object — file + metadata. Max 5 TB per object.

Key — full path to the object (acts like filename)

Versioning — keep all versions of every object

Durability: 99.999999999% (11 9s)

🗂️ S3 Storage Classes

Standard — $0.023/GB. Hot data, frequent access.

Standard-IA — $0.0125/GB. Monthly access.

Intelligent-Tiering — Auto moves between tiers.

Glacier Instant — $0.004/GB. Quarterly access.

Deep Archive — $0.00099/GB. Yearly access.

🔑 IAM — Identity & Access Management

Golden Rule of IAM Grant the least privilege — only the permissions needed to do the job. Nothing more.
👤 Users

A person or application. Has credentials (password + access keys). Avoid using root account for daily tasks.

👥 Groups

Collection of users. Attach policies to groups, not individual users. Users inherit group permissions.

🎭 Roles

Assumed by services (EC2, Lambda) or users. Temporary credentials. No long-term access keys. Best for service-to-service access.

📋 Policies

JSON document defining Allow/Deny for Actions on Resources. Identity-based (attached to user/group/role) or Resource-based (attached to S3 bucket, etc.).

🔐 MFA

Enable Multi-Factor Authentication on root and all IAM users. Required for sensitive actions and console login. Use hardware MFA for root.

🌐 Federation

Login with corporate identity (AD, Google). Assumes IAM Role via SAML 2.0, OIDC, or AWS SSO / IAM Identity Center.

🌐 VPC Basics

🏗️ VPC Components

VPC — your private network in AWS (e.g. 10.0.0.0/16)

Subnet — sub-range of VPC. Public (internet) or Private.

Internet Gateway — allows public subnet → internet

NAT Gateway — allows private subnet → internet (not reverse)

Route Table — defines where traffic goes

Security Group — stateful firewall on instance level

NACL — stateless firewall on subnet level

🏛️ 3-Tier Architecture in VPC

Public Subnet — ALB, Bastion (or use Session Manager)

Private Subnet (App) — EC2/ECS, Lambda

Private Subnet (DB) — RDS, ElastiCache

Internet → ALB → App → DB. Only ALB faces internet. App and DB have no public IPs.

📘 INTERMEDIATE LEVEL — Core Services Deep Dive

⚡ Lambda — Serverless Functions

🔑 Key Concepts

Trigger — what invokes the function (API Gateway, S3, SQS, EventBridge, etc.)

Handler — entry point function in your code

Execution Role — IAM role the function assumes

Timeout — max 15 minutes. Default 3 seconds.

Memory — 128 MB to 10 GB. CPU scales with memory.

❄️ Cold Start vs Warm

Cold start — first invocation after idle. AWS provisions container + loads runtime + loads code. Adds 100ms–2s latency.

Warm — reuses existing container. Near-instant.

Mitigation: Provisioned Concurrency (keeps N instances warm), smaller deployment packages, avoid VPC if not needed.

⚡ Concurrency

Concurrent executions — one per simultaneous request. Default limit: 1000/account/region.

Reserved concurrency — cap a function's max concurrent executions.

Provisioned concurrency — pre-warm N instances to eliminate cold starts.

🔄 Invocation Types

Synchronous — caller waits for result. API Gateway, ALB.

Asynchronous — fire-and-forget. S3, SNS, EventBridge. Built-in retry (2×).

Poll-based — Lambda polls SQS/Kinesis/DynamoDB Streams. Batch processing.

📨 Messaging — SNS, SQS, EventBridge

📢 SNS — Simple Notification Service

Pub/Sub. Publisher sends to a Topic. All subscribers receive the message. Push model.

Subscribers: SQS, Lambda, HTTP, Email, SMS, mobile push.

Fan-out pattern — one message → multiple queues

📬 SQS — Simple Queue Service

Message queue. Producers send messages, consumers poll and process. Decouples services.

Standard: at-least-once, best-effort order.

FIFO: exactly-once, strict order. 3000 msg/s.

DLQ — dead letter queue for failed messages

⚡ EventBridge

Serverless event bus. Route events from AWS services, SaaS, or your apps to targets.

Rules: match event patterns, schedule via cron.

Targets: Lambda, SQS, SNS, ECS, Step Functions, API GW.

Schema Registry — discover event shapes

🐳 Containers — ECS & EKS

🐳 ECS — Elastic Container Service

AWS-native container orchestration. Two launch types:

EC2 — you manage the EC2 instances (servers). More control.

Fargate — serverless. AWS manages servers. Just define CPU/memory.

Task Definition → Service → Cluster. ALB integrates natively.

⚓ EKS — Elastic Kubernetes Service

Managed Kubernetes. Run standard K8s workloads on AWS.

Use when: team already knows K8s, need multi-cloud portability, complex scheduling.

Add-ons: AWS Load Balancer Controller, EBS CSI, VPC CNI, Karpenter (node autoscaling).

🏗️ Infrastructure as Code

☁️ CloudFormation

AWS-native IaC. JSON or YAML templates. Stacks manage resource lifecycle. Change sets preview changes before applying.

🏗️ CDK — Cloud Development Kit

Write infrastructure in TypeScript/Python/C#/Java. Compiles to CloudFormation. Higher-level constructs (L1/L2/L3).

🌿 Terraform

Multi-cloud IaC. HCL language. State file tracks resources. Best for multi-cloud or existing TF teams.

🔥 ADVANCED LEVEL — Architecture Patterns

🏛️ Architecture Patterns

🎯 Microservices

Split monolith into small, independent services. Each owns its data. Communicate via REST, gRPC, or async messaging (SNS/SQS/EventBridge).

AWS pattern: API Gateway → Lambda/ECS (per service) → own RDS/DynamoDB table → SNS/SQS for inter-service events.

Bounded Context Domain-Driven Design

⚡ Event-Driven Architecture

Services communicate by publishing events. Producers don't know consumers. Loose coupling, high scalability.

Pattern: Order placed → EventBridge → parallel: inventory update + payment processing + email notification.

Saga Pattern for distributed transactions

🔄 CQRS + Event Sourcing

CQRS: Separate read (Query) and write (Command) models. Write to DynamoDB, read from ElastiCache or OpenSearch.

Event Sourcing: Store events, not state. Replay events to rebuild state. DynamoDB Streams + Kinesis.

🏗️ Strangler Fig Pattern

Migrate monolith to microservices gradually. Route traffic via API Gateway — old paths go to monolith, new paths go to new services. Strangle the monolith piece by piece.

🌊 Streaming — Kinesis

📊 Kinesis Data Streams

Real-time data streaming. Shards define throughput (1 MB/s in, 2 MB/s out per shard). Data retained 1–365 days. Consumers: Lambda, Kinesis Analytics, custom.

Enhanced Fan-Out — 2 MB/s per consumer per shard via HTTP/2 push

🔥 Kinesis Data Firehose

Managed delivery to S3, Redshift, OpenSearch, Splunk. Auto-scales. Transform with Lambda. No consumer management needed.

Near real-time — 60s buffer window minimum

🔁 Step Functions

Orchestrate workflows visually Step Functions lets you define state machines with states: Task, Choice, Wait, Parallel, Map, Pass, Fail, Succeed. Each step calls a Lambda, ECS task, DynamoDB, SNS, SQS, API GW, or other AWS service.
⚡ Express Workflows

For high-volume, short-duration workloads (<5 min). At-least-once. Up to 100K executions/sec. Good for real-time streaming processing.

📋 Standard Workflows

For long-running, auditable workflows (up to 1 year). Exactly-once. Max 2000 executions/sec. Good for order processing, approval flows.

🏎️ Performance & Caching

⚡ ElastiCache (Redis)

Cache database results. Session storage. Rate limiting. Leaderboards. Sub-ms reads. Reduces DB load by 90%+.

Cache-Aside pattern: check cache → miss → DB → write cache

🌐 CloudFront CDN

Cache static + dynamic content at 400+ edge locations. Reduce latency for global users. Integrates with S3, ALB, API GW, Lambda@Edge.

Lambda@Edge — run code at edge locations

⚡ DynamoDB DAX

In-memory cache for DynamoDB. Read latency from ms to microseconds. Fully compatible API — no code changes needed. Good for read-heavy workloads.

⭐ EXPERT LEVEL — Production & Enterprise Patterns

🏢 Multi-Account Strategy

Why multiple accounts? Accounts are the strongest security boundary in AWS. Blast radius isolation, separate billing, independent limits, different security postures per environment.
🏢 AWS Organizations

Group accounts into Organizational Units (OUs). Apply Service Control Policies (SCPs) at OU level. Consolidated billing. One management account, N member accounts.

Typical structure: Root → Security OU → Workloads OU → Production account + Staging account + Dev account

🗼 AWS Control Tower

Automated landing zone setup. Enforces guardrails (preventive via SCPs, detective via Config rules). Account Factory provisions new accounts with standards pre-applied.

Guardrails: Mandatory (always on), Strongly Recommended, Elective.

📋 Service Control Policies (SCPs)

IAM-like JSON policies attached to OUs. Limit what member accounts can do — even if they have AdministratorAccess. Cannot grant permissions, only restrict.

Example: Deny creating resources outside approved regions. Deny disabling CloudTrail. Deny leaving Organization.

🔐 AWS IAM Identity Center

Formerly AWS SSO. Single sign-on to multiple accounts with one set of credentials. Integrates with AD, Okta, Google. Permission sets map to IAM roles per account.

💰 Cost Optimisation

📊 Cost Visibility

Cost Explorer — visualise, filter, forecast spending.

Cost & Usage Report — hourly granular data to S3 + Athena.

Budgets — alert when spend exceeds threshold.

Tagging strategy — tag every resource with App, Env, Team, CostCentre.

⚙️ Compute Savings

Right-size EC2 with Compute Optimizer recommendations. Move to Graviton (ARM) — 40% better price/perf. Spot for batch/fault-tolerant. Savings Plans for baseline.

Graviton3 — up to 40% cost saving vs x86

🗄️ Storage Savings

S3 Intelligent-Tiering for auto-optimised access. Lifecycle policies → Glacier. EBS: delete unattached volumes + old snapshots. RDS storage autoscaling.

S3 Lens — storage analytics and recommendations

🌊 Chaos Engineering & Resilience

💥 AWS Fault Injection Service (FIS)

Inject controlled failures — terminate EC2, throttle API calls, inject network latency, kill RDS. Run experiments in staging to validate resilience before incidents happen.

Experiment Templates define actions, targets, stop conditions

🛡️ Resilience Best Practices

✅ Multi-AZ for all stateful resources

✅ Circuit breakers in service calls

✅ Exponential backoff + jitter for retries

✅ Dead letter queues for all async processing

✅ Health checks with proper thresholds

✅ Graceful degradation (return cached/default data)

🚀 Advanced Networking

🔗 Transit Gateway

Hub-and-spoke model. Connect multiple VPCs + on-prem via single gateway. Replaces complex VPC peering mesh. Supports inter-region peering.

⚡ Direct Connect

Dedicated private network from on-prem to AWS. 1 Gbps to 100 Gbps. Lower latency, consistent bandwidth, no internet. For hybrid workloads.

🌐 PrivateLink

Expose services privately to other VPCs/accounts without traversing internet. Used by AWS services internally. Build private SaaS. No VPC peering needed.

📏 Scalability Patterns

HORIZONTAL vs VERTICAL SCALING:

Vertical (Scale Up):
  t3.micro → t3.xlarge → m6i.4xlarge
  Limit: Largest instance has a ceiling. Single point of failure.

Horizontal (Scale Out):
  1 instance → 10 instances → 1000 instances
  Requires: Stateless app (session in ElastiCache, not local) + ALB
  Pattern: Auto Scaling Group + ALB + Target Tracking (CPU 70%)

SHARDING (Database horizontal scale):
  DynamoDB → auto-shards by partition key
  RDS → read replicas for read scale, manual sharding for write scale
  Aurora → up to 15 read replicas, Serverless v2 auto-scales ACUs

🔒 Security Deep Dive — Defence in Depth
Shared Responsibility Model AWS is responsible for security OF the cloud (hardware, datacentres, networking). You are responsible for security IN the cloud (OS, data, IAM, encryption, network config).

🔑 Identity & Access

🔐 KMS — Key Management Service

Create and manage encryption keys. CMKs (Customer Managed Keys) for full control. Automatic annual rotation. Integrates with S3, RDS, EBS, Lambda, DynamoDB, Secrets Manager.

Envelope encryption — KMS encrypts a data key, data key encrypts data

🗝️ Secrets Manager

Store and auto-rotate secrets (DB passwords, API keys). Lambda-based rotation. Cross-account sharing. Audit access via CloudTrail. $0.40/secret/month.

Auto-rotation — rotate every N days without app restart

🔏 Certificate Manager (ACM)

Free SSL/TLS certificates for ALB, CloudFront, API Gateway. Auto-renewal. Cannot export private key. Use AWS Private CA for internal services.

🛡️ Threat Detection & Protection

🔍 GuardDuty

Managed threat detection. Analyses CloudTrail, VPC Flow Logs, DNS logs. Detects: crypto-mining, credential exfiltration, unusual API calls, port scanning. AI-driven, zero setup.

🛡️ WAF — Web Application Firewall

Protect ALB, CloudFront, API Gateway from SQL injection, XSS, OWASP Top 10. Rate limiting. IP blocking. Managed rule groups from AWS and marketplace.

🏰 Shield

Standard: Free. Automatic L3/L4 DDoS protection for all AWS resources.
Advanced: $3000/month. L7 DDoS, 24/7 DRT team, cost protection, global threat dashboard.

🔎 Security Hub

Central security posture view. Aggregates findings from GuardDuty, Inspector, Macie, IAM Access Analyser. Benchmarks: CIS AWS, PCI DSS, AWS Foundational Security.

🕵️ Macie

AI-powered sensitive data discovery in S3. Detects PII (names, credit cards, SSNs), financial data, credentials. Alerts on public or unencrypted buckets with sensitive data.

📋 AWS Config

Track resource configuration changes over time. Evaluate against rules (Is S3 bucket public? Is MFA on root enabled?). Remediate automatically via SSM Automation.

🧠 Knowledge Quiz — Test Yourself!

Click an answer to see if you're right. Covers all levels — beginner to expert.

Q1 🌱 Which S3 storage class offers the LOWEST storage cost?
A S3 Standard
B S3 Glacier Instant Retrieval
C S3 Glacier Flexible Retrieval
D S3 Glacier Deep Archive
✅ Deep Archive at ~$0.00099/GB/month is the cheapest storage on AWS — 23× cheaper than S3 Standard ($0.023/GB). Used for 7–10 year regulatory archives.
Q2 📘 A Lambda function needs to read from DynamoDB without hardcoding credentials. What should you use?
A Store access keys in environment variables
B Attach an IAM execution role to the Lambda function
C Hardcode credentials in the Lambda code
D Use a root account access key
✅ IAM Execution Role — Lambda assumes the role at runtime for temporary credentials automatically via the AWS SDK. Never hardcode credentials or use root account keys.
Q3 📘 You need to decouple a microservice so that if the downstream service is slow, the upstream isn't affected. Which service?
A SNS
B SQS
C EventBridge
D API Gateway
✅ SQS is the classic decoupling pattern. The upstream writes to the queue and returns immediately. The downstream consumes at its own pace. If the downstream is down, messages stay safely in the queue.
Q4 🔥 Your app needs the lowest possible RTO and RPO for disaster recovery. Which strategy?
A Backup and Restore
B Pilot Light
C Warm Standby
D Multi-Site Active/Active
✅ Multi-Site Active/Active has near-zero RTO and RPO. Full capacity runs simultaneously in 2+ regions. Route 53 health checks instantly fail over. Most expensive but most resilient.
Q5 🔥 Which service prevents deletion of objects in S3 — even by the AWS root account?
A S3 Bucket Policy with Deny
B S3 Object Lock (Governance Mode)
C S3 Object Lock (Compliance Mode)
D MFA Delete
✅ Compliance Mode — once set, nobody (not even root) can delete the object until the retention date. Used for SEC 17a-4, FINRA, HIPAA compliance. Governance mode allows admin override.
Q6 ⭐ Which AWS service enforces permission boundaries across an entire AWS Organisation — even overriding AdministratorAccess?
A IAM Permission Boundaries
B Service Control Policies (SCPs)
C AWS Config Rules
D IAM Role Trust Policies
✅ SCPs (Service Control Policies) in AWS Organizations apply to entire OUs. They restrict what IAM identities in member accounts can do — even if they have AdministratorAccess. SCPs cannot grant permissions, only restrict.
Q7 📘 Which EC2 pricing model offers up to 90% discount but can be interrupted by AWS?
A Reserved Instances
B Savings Plans
C Spot Instances
D Dedicated Hosts
✅ Spot Instances use unused EC2 capacity at up to 90% discount. AWS can reclaim them with 2-minute notice when capacity is needed. Use for fault-tolerant batch jobs, ML training, stateless services.
Q8 🔥 A DynamoDB table is experiencing hot partition issues. What's the best fix?
A Increase provisioned capacity
B Switch to On-Demand billing
C Choose a higher-cardinality partition key
D Enable DynamoDB Streams
✅ Higher-cardinality partition key — hot partitions happen when too many requests go to the same partition. A more unique key (e.g. orderId vs status) distributes traffic evenly. Also consider adding a random suffix for write-heavy partitions.
