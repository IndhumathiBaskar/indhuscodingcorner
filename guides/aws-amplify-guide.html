⚡ What is AWS Amplify?

AWS Amplify is a complete platform for building and deploying full-stack web and mobile applications on AWS. It bundles Amplify Hosting (deploy frontend apps with CI/CD from Git), Amplify Backend (provision Auth, API, Storage, Functions via CLI or Gen 2 CDK), and a set of client-side libraries that connect your React/Next.js/Vue app to those backends in a few lines of code.

💡 Real-life analogy — The Furnished Apartment

Building a web app from scratch on AWS is like buying an empty plot and constructing a house — you pick every brick (EC2, RDS, Cognito, S3, CloudFront…). Amplify is the fully furnished apartment. Move in your code and it wires up authentication, a database API, file storage, and a global CDN for you. The furniture (Cognito, AppSync, DynamoDB, S3, CloudFront) is standard AWS under the hood — Amplify just places it, arranges it, and connects everything so you don't spend weeks on plumbing before writing your first feature.

📡 Data & Storage — AppSync GraphQL + S3

Amplify Data provisions an AWS AppSync GraphQL API backed by DynamoDB from a simple TypeScript schema. You define your models — Amplify generates the DynamoDB tables, resolvers, and full CRUD GraphQL operations automatically. Amplify Storage gives your users a per-user, per-group S3 bucket with fine-grained access — no S3 policy expertise needed.

Data Model Definition (Gen 2)

TypeScript — amplify/data/resource.ts
import { a, defineData, ClientSchema } from '@aws-amplify/backend';

const schema = a.schema({
  // ── DynamoDB table + AppSync CRUD operations ──────────────
  Todo: a
    .model({
      title:     a.string().required(),
      content:   a.string(),
      done:      a.boolean().default(false),
      priority:  a.enum(['LOW', 'MEDIUM', 'HIGH']),
      owner:     a.string(),
    })
    .authorization(allow => [
      allow.owner(),             // only creator can CRUD
      allow.authenticated().to(['read']) // all auth users can read
    ]),

  Post: a
    .model({
      title:   a.string().required(),
      body:    a.string(),
      author:  a.belongsTo('User', 'authorId'),
    })
    .authorization(allow => [allow.authenticated()]),
});

export type Schema = ClientSchema<typeof schema>;

export const data = defineData({
  schema,
  authorizationModes: {
    defaultAuthorizationMode: 'userPool', // Cognito JWT
  },
});

Amplify Storage — S3 with Access Rules

TypeScript — amplify/storage/resource.ts + frontend upload
// ── Backend: define access rules ───────────────────────────
import { defineStorage } from '@aws-amplify/backend';

export const storage = defineStorage({
  name: 'myAppStorage',
  access: (allow) => ({
    // Each user gets their own private "folder"
    'profile-pictures/{entity_id}/*': [
      allow.entity('identity').to(['read', 'write', 'delete']),
      allow.authenticated().to(['read']), // all users can read profiles
    ],
    // Admin-only uploads area
    'shared-assets/*': [
      allow.group('Admin').to(['read', 'write']),
      allow.authenticated().to(['read']),
    ],
  }),
});

// ── Frontend: upload a file ────────────────────────────────
import { uploadData, getUrl } from 'aws-amplify/storage';

await uploadData({
  path: `profile-pictures/${userId}/avatar.jpg`,
  data: file,                // File from input[type=file]
  options: { contentType: 'image/jpeg' },
});

// ── Get a temporary signed URL to display the image ───────
const { url } = await getUrl({
  path: `profile-pictures/${userId}/avatar.jpg`,
});

Complete amplify.yml — Next.js with Caching

YAML — amplify.yml (Next.js, with caching + env branches)
version: 1

applications:
  - frontend:
      phases:
        preBuild:
          commands:
            - npm ci                      # clean install (faster than npm install)
        build:
          commands:
            - npm run build
        postBuild:
          commands:
            - echo "Build complete for branch $AWS_BRANCH"
      artifacts:
        baseDirectory: .next         # Next.js output folder
        files:
          - '**/*'
      cache:
        paths:
          - node_modules/**/*          # cache node_modules between builds
          - .next/cache/**/*           # cache Next.js build cache
    appRoot: frontend              # monorepo: app lives in /frontend subfolder

Monorepo Setup — Multiple Apps in One Repo

version: 1
applications:

💡 Always use npm ci not npm install in builds

npm ci installs exactly from package-lock.json — no version drift between developers and CI. It's also faster in CI because it deletes node_modules first and doesn't update the lockfile. npm install in CI can silently install a newer patch version of a dependency that breaks your build — npm ci prevents this entirely.

💻 C# Backend — Integrating with Amplify APIs

Amplify's frontend libraries target JavaScript/TypeScript. But your .NET API backend integrates with Amplify's outputs: validate Cognito JWTs in ASP.NET Core, call AppSync from C# with HttpClient, and read Amplify environment outputs from the amplify_outputs.json configuration file that Amplify generates per deployment.

Validate Cognito JWT in ASP.NET Core

C# — Program.cs — Cognito JWT validation middleware
using Microsoft.AspNetCore.Authentication.JwtBearer;
using Microsoft.IdentityModel.Tokens;

var builder = WebApplication.CreateBuilder(args);

// ── Cognito JWT validation ─────────────────────────────────
var region   = builder.Configuration["Cognito:Region"];     // ap-south-1
var userPool = builder.Configuration["Cognito:UserPoolId"]; // ap-south-1_xxxxxxx
var authority = $"https://cognito-idp.{region}.amazonaws.com/{userPool}";

builder.Services
    .AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.Authority = authority;
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuerSigningKey = true,
            ValidateIssuer           = true,
            ValidateAudience         = false, // Cognito access tokens have no aud
            ValidIssuer              = authority
        };
    });

builder.Services.AddAuthorization();

var app = builder.Build();
app.UseAuthentication();
app.UseAuthorization();

Read Authenticated User Claims in Controller

C# — extract Cognito user info from JWT claims
[ApiController]
[Route("api/[controller]")]
[Authorize]  // requires valid Cognito JWT
public class ProfileController : ControllerBase
{
    [HttpGet]
    public IActionResult GetProfile()
    {
        // Cognito JWT standard claims
        var userId   = User.FindFirst("sub")?.Value;       // unique Cognito user ID
        var email    = User.FindFirst("email")?.Value;
        var username = User.FindFirst("cognito:username")?.Value;

        // Cognito groups (for role-based access)
        var groups   = User.Claims
            .Where(c => c.Type == "cognito:groups")
            .Select(c => c.Value)
            .ToList();

        var isAdmin  = groups.Contains("Admin");

        return Ok(new { userId, email, username, groups, isAdmin });
    }

    [HttpGet("admin-only")]
    [Authorize(Policy = "AdminOnly")]
    public IActionResult AdminEndpoint()
        => Ok("You are an Admin!");
}

Call AppSync GraphQL from C#

C# — AppSync GraphQL client using HttpClient
public class AppSyncClient
{
    private readonly HttpClient _http;
    private readonly string _endpoint;  // from amplify_outputs.json → data.url
    private readonly string _apiKey;    // from amplify_outputs.json → data.api_key

    public AppSyncClient(HttpClient http, IConfiguration cfg)
    {
        _http     = http;
        _endpoint = cfg["AppSync:Endpoint"]!;
        _apiKey   = cfg["AppSync:ApiKey"]!;
    }

    // ── Execute any GraphQL operation ──────────────────────────
    public async Task<T?> QueryAsync<T>(
        string query,
        object? variables = null)
    {
        var payload = JsonSerializer.Serialize(new
        {
            query,
            variables
        });

        var request = new HttpRequestMessage(HttpMethod.Post, _endpoint)
        {
            Content = new StringContent(payload, Encoding.UTF8, "application/json")
        };
        request.Headers.Add("x-api-key", _apiKey);  // or use Cognito JWT instead

        var resp = await _http.SendAsync(request);
        resp.EnsureSuccessStatusCode();

        using var doc = await JsonDocument.ParseAsync(
            await resp.Content.ReadAsStreamAsync());

        return doc.RootElement
            .GetProperty("data")
            .Deserialize<T>();
    }
}

✅ Best Practices & Common Mistakes

Hosting & Deployment

  • Always use npm ci instead of npm install in your amplify.yml — ensures reproducible builds from package-lock.json
  • Cache node_modules and framework build caches (e.g. .next/cache) in amplify.yml — cuts build time by 50–70%
  • Pin your Node.js version with a .nvmrc file or NODE_VERSION env var — prevents framework incompatibility surprises
  • Use branch-specific environment variables for API URLs, feature flags, and analytics IDs — never hardcode environment differences in code
  • Enable PR previews for all teams — stakeholders reviewing a real URL prevents dozens of "can you show me?" meetings
  • Test instant rollback before you need it — redeploy a previous version and confirm it works under load

Auth & Security

  • Never put secrets or API keys in NEXT_PUBLIC_ env vars — they are baked into the client bundle and visible to anyone
  • Store sensitive build-time secrets in AWS Secrets Manager and reference via Amplify's secret() function in Gen 2
  • Enable MFA (TOTP at minimum) for all admin Cognito user groups
  • Validate Cognito JWTs in your .NET API — check issuer, signature, and expiry — never trust the frontend alone
  • Scope AppSync authorization per model — don't use allow.public() for data models that contain user data
  • Enable Cognito advanced security (threat protection) for production apps — detects credential stuffing and account takeover

Data & Storage

  • Use allow.owner() authorization for user-specific data — prevents one user reading another user's records
  • Never expose raw S3 URLs — always use Amplify's getUrl() which returns short-lived signed URLs
  • Set S3 lifecycle rules on your Amplify Storage bucket for temporary uploads — delete files older than N days to control costs
  • Use real-time subscriptions (observeQuery) only where live updates are truly needed — each subscription is a WebSocket connection, and too many hurt mobile battery life

Common Mistakes to Avoid

⚠️ Top Amplify Mistakes

❌ Putting secrets in NEXT_PUBLIC_ variables — these are bundled into the JavaScript served to browsers; anyone can read them in DevTools
❌ Using npm install in builds instead of npm ci — causes non-reproducible builds; different package versions in CI vs local
❌ Not caching node_modules in amplify.yml — 3–5 minute package install on every single build; cache cuts it to seconds
❌ Forgetting to validate Cognito JWT in your .NET API — the frontend Amplify library handles auth for the client, but your backend must independently validate every token
❌ Using allow.public() on data models containing PII — makes data readable by anyone without authentication, including bots
❌ Leaving open PR preview environments running forever — each branch environment costs money; delete merged/closed PR branches to clean up
❌ Not pinning Node.js version — Amplify's default Node version changes; a platform upgrade can silently break your build
❌ Mixing Gen 1 (CLI) and Gen 2 (CDK) backend approaches — they use different deployment models; pick Gen 2 for all new projects
