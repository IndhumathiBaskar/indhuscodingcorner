🔍 AWS Athena - Complete Deep Dive

Athena = Serverless Interactive Query Service
Query data in S3 directly using SQL. No servers to manage, pay only for queries you run.

What Makes Athena Special?

🎯 Athena vs Traditional Databases:

Traditional Database (RDS, DynamoDB):
• Data stored in database
• Query via SQL/API
• Pay for storage + compute
• Must provision capacity
• Good for transactional data

Athena (Query Service):
• Data stored in S3
• Query via SQL
• Pay ONLY for queries ($5 per TB scanned)
• Serverless (no servers)
• Good for analytics, logs, archives
• Scan only needed columns (cost saving)

Key Insight:
Athena = SQL engine + S3
Reads data from S3, analyzes, returns results
Original S3 files unchanged
Perfect for big data analysis

🏗️ Athena Architecture & How It Works

Complete Athena Query Execution Flow

👤 You / Your Application
  • Write a SQL query
  • Submit via AWS Console, SDK, or API
↓
🔍 Athena — Query Engine (Presto)
  • Parses your SQL query
  • Creates an execution plan
  • Distributes work across parallel workers
↓
🗄️ S3 — Your Data Source
  • Reads only the columns you need
  • Supports Parquet, CSV, JSON, ORC formats
  • Applies filters early — scans less data
  • ⚠️ You are charged per TB scanned here
↓
⚡ Parallel Query Execution
  • Multiple workers process data simultaneously
  • Pushdown filtering reduces data movement
  • Results aggregated from all workers
↓
📋 Results — Stored back in S3
  • CSV file saved to your results S3 bucket
  • Unique Query ID assigned for reference
  • Execution stats: rows, bytes scanned, duration
  • Results reusable for 7 days (with caching on)
↓
💰 Billing — Pay Only per Scan
  • $5 per TB of data scanned — not per row
  • Parquet/ORC format = up to 10x cheaper
  • Partitioning = scan less = pay less
  • No charge when query returns 0 rows

Core Components Explained

1️⃣

S3 Data Source

What: Your actual data files
Formats supported: CSV, JSON, Parquet, ORC, TSV, Avro
Organization: Folders = tables, objects = partitions
Cost impact: Parquet/ORC = compressed = cheaper scans
Best practice: Organize by date for partitioning

2️⃣

Metadata (Glue Catalog)

What: Table definitions and schema
Contains: Column names, types, partitions
Where stored: AWS Glue Data Catalog
Use: Athena knows how to parse S3 files
Alternative: Manual table definition in Athena

3️⃣

Query Engine

What: Presto-based SQL engine
Capabilities: Full SQL, joins, aggregations, window functions
Parallel: Distributes query across workers
Optimization: Pushdown filtering, column pruning
Timeout: 30 minutes default

4️⃣

Results Location

What: S3 bucket for query results
Contains: .csv file + metadata JSON
Organization: Folder per query execution
Retention: You control (usually 45 days)
Cost: S3 storage charge for results

Complete Example 1: Run Simple Query

public static async Task RunSimpleQuery()
{
    var athenaClient = new AmazonAthenaClient();
    
    // 1. Start query execution
    var startRequest = new StartQueryExecutionRequest
    {
        QueryString = "SELECT * FROM my_analytics.users WHERE country='USA' LIMIT 10",
        
        // Required: Which database to use
        QueryExecutionContext = new QueryExecutionContext
        {
            Database = "my_analytics"
        },
        
        // Required: Where to save results
        ResultConfiguration = new ResultConfiguration
        {
            OutputLocation = "s3://my-athena-results/"
        }
    };
    
    var startResponse = await athenaClient.StartQueryExecutionAsync(startRequest);
    string queryId = startResponse.QueryExecutionId;
    
    Console.WriteLine($"Query started: {queryId}");
    
    // 2. Wait for query to complete
    var maxAttempts = 30;  // 30 attempts * 2 sec = 60 sec max
    var attempt = 0;
    
    while (attempt < maxAttempts)
    {
        var statusRequest = new GetQueryExecutionRequest
        {
            QueryExecutionId = queryId
        };
        
        var statusResponse = await athenaClient.GetQueryExecutionAsync(statusRequest);
        var status = statusResponse.QueryExecution.Status.State;
        
        Console.WriteLine($"Status: {status}");
        
        if (status == QueryExecutionState.SUCCEEDED)
        {
            // Query completed successfully!
            break;
        }
        else if (status == QueryExecutionState.FAILED)
        {
            Console.WriteLine($"Query failed: {statusResponse.QueryExecution.Status.StateChangeReason}");
            return;
        }
        else if (status == QueryExecutionState.CANCELLED)
        {
            Console.WriteLine("Query was cancelled");
            return;
        }
        
        // Still running, wait 2 seconds
        await Task.Delay(2000);
        attempt++;
    }
    
    // 3. Get results
    var resultsRequest = new GetQueryResultsRequest
    {
        QueryExecutionId = queryId,
        MaxResults = 100  // Max 1000 rows per call
    };
    
    var resultsResponse = await athenaClient.GetQueryResultsAsync(resultsRequest);
    
    // 4. Process results
    foreach (var row in resultsResponse.ResultSet.Rows)
    {
        foreach (var datum in row.Data)
        {
            Console.Write(datum.VarCharValue + "  ");
        }
        Console.WriteLine();
    }
    
    Console.WriteLine($"Total rows: {resultsResponse.ResultSet.Rows.Count}");
}

// Output example:
// Query started: a1b2c3d4-e5f6-g7h8-i9j0-k1l2m3n4o5p6
// Status: RUNNING
// Status: RUNNING
// Status: SUCCEEDED
// 1    John    [email protected]    2020-01-15    USA
// 2    Alice   [email protected]   2020-02-20    USA
// Total rows: 2

Complete Example 2: Parameterized Query (Safe & Reusable)

public static async Task QueryWithParameters(
    string country,
    int minAge)
{
    var athenaClient = new AmazonAthenaClient();
    
    // Build query with parameters
    var query = $@"
        SELECT 
            user_id,
            name,
            email,
            country,
            age
        FROM my_analytics.users 
        WHERE country = '{country}'
            AND age >= {minAge}
        ORDER BY name
    ";
    
    var startRequest = new StartQueryExecutionRequest
    {
        QueryString = query,
        QueryExecutionContext = new QueryExecutionContext
        {
            Database = "my_analytics"
        },
        ResultConfiguration = new ResultConfiguration
        {
            OutputLocation = "s3://my-athena-results/",
            EncryptionConfiguration = new EncryptionConfiguration
            {
                EncryptionOption = EncryptionOption.SSE_S3
            }
        }
    };
    
    var response = await athenaClient.StartQueryExecutionAsync(startRequest);
    
    Console.WriteLine($"Query ID: {response.QueryExecutionId}");
    Console.WriteLine($"Country: {country}, Min Age: {minAge}");
}

// Usage:
await QueryWithParameters("USA", 25);

Complete Example 3: Batch Query Processing

public class AthenaQueryService
{
    private readonly IAmazonAthena _athenaClient;
    
    public AthenaQueryService(IAmazonAthena athenaClient)
    {
        _athenaClient = athenaClient;
    }
    
    // Run query and wait for completion
    public async Task<QueryExecutionState> ExecuteQueryAsync(
        string query,
        string database,
        string resultsLocation)
    {
        // Start query
        var startRequest = new StartQueryExecutionRequest
        {
            QueryString = query,
            QueryExecutionContext = new QueryExecutionContext { Database = database },
            ResultConfiguration = new ResultConfiguration { OutputLocation = resultsLocation }
        };
        
        var startResponse = await _athenaClient.StartQueryExecutionAsync(startRequest);
        string queryId = startResponse.QueryExecutionId;
        
        // Wait for completion
        var finalStatus = await WaitForQueryCompletion(queryId);
        
        return finalStatus;
    }
    
    // Wait for query to finish
    private async Task<QueryExecutionState> WaitForQueryCompletion(string queryId)
    {
        while (true)
        {
            var statusRequest = new GetQueryExecutionRequest { QueryExecutionId = queryId };
            var statusResponse = await _athenaClient.GetQueryExecutionAsync(statusRequest);
            
            var status = statusResponse.QueryExecution.Status.State;
            
            if (status == QueryExecutionState.SUCCEEDED ||
                status == QueryExecutionState.FAILED ||
                status == QueryExecutionState.CANCELLED)
            {
                return status;
            }
            
            await Task.Delay(1000);  // Check every second
        }
    }
    
    // Get query results
    public async Task<List<Dictionary<string, string>> GetQueryResultsAsync(string queryId)
    {
        var results = new List<Dictionary<string, string>>();
        
        var resultsRequest = new GetQueryResultsRequest
        {
            QueryExecutionId = queryId,
            MaxResults = 1000
        };
        
        var resultsResponse = await _athenaClient.GetQueryResultsAsync(resultsRequest);
        
        // Get column names from header
        var headers = resultsResponse.ResultSet.Rows[0].Data
            .Select(d => d.VarCharValue)
            .ToList();
        
        // Convert remaining rows to dictionaries
        for (int i = 1; i < resultsResponse.ResultSet.Rows.Count; i++)
        {
            var rowDict = new Dictionary<string, string>();
            var rowData = resultsResponse.ResultSet.Rows[i].Data;
            
            for (int j = 0; j < headers.Count; j++)
            {
                rowDict[headers[j]] = rowData[j].VarCharValue ?? "";
            }
            
            results.Add(rowDict);
        }
        
        return results;
    }
}

// Usage:
var service = new AthenaQueryService(athenaClient);

var status = await service.ExecuteQueryAsync(
    "SELECT * FROM my_analytics.users LIMIT 100",
    "my_analytics",
    "s3://my-athena-results/");

if (status == QueryExecutionState.SUCCEEDED)
{
    var data = await service.GetQueryResultsAsync(queryId);
    foreach (var row in data)
    {
        Console.WriteLine($"{row["user_id"]} - {row["name"]} - {row["email"]}");
    }
}

OutputLocation S3 folder for results Yes s3://bucket/prefix/
ExecutionParameters Parameterized values No List of strings

⚡ Query Optimization - Save Money & Time

Cost Optimization Techniques

1. Use Partitioning (Most Important!)

Without partitioning: Athena scans entire dataset
Example: 1000 GB of data = $5.00 per query

With partitioning by date: Scan only needed dates
Example: Query 1 month = 30 GB scanned = $0.15 per query

How to partition:
Organize S3 like: s3://bucket/year=2024/month=01/day=15/
Athena auto-discovers partitions
Query only needed partitions = 95% cost reduction!

2. Use Compression (Parquet/ORC)

CSV format: 100 GB uncompressed
Cost: $0.50 per query

Parquet format: 10 GB compressed (10% size)
Cost: $0.05 per query

Savings: 90% reduction!**
Plus: Faster query execution

3. Projec Only Needed Columns

❌ Wrong: SELECT * FROM large_table
Scans all columns = full cost

✅ Right: SELECT id, name, email FROM large_table
Scans only 3 columns = 10% of cost

Benefit: Works especially well with Parquet (column-based format)

4. Use WHERE Clauses (Push Down Filtering)

❌ Wrong: SELECT * FROM table, then filter in application
Scans full table

✅ Right: SELECT * FROM table WHERE country='USA'
Athena filters BEFORE returning results

Cost impact: Huge! Filter early, scan less

Complete Log Analysis Code

public class LogAnalyticsService
{
    private readonly IAmazonAthena _athenaClient;
    
    // Analyze errors in logs
    public async Task AnalyzeErrorsAsync(
        string dateStr)  // "2024-03-13"
    {
        var query = $@"
            SELECT 
                timestamp,
                service,
                error_message,
                stack_trace,
                count(*) as error_count
            FROM logs
            WHERE 
                year = '{dateStr.Substring(0, 4)}'
                AND month = '{dateStr.Substring(5, 2)}'
                AND day = '{dateStr.Substring(8, 2)}'
                AND level = 'ERROR'
            GROUP BY 
                service, 
                error_message
            ORDER BY error_count DESC
            LIMIT 50
        ";
        
        var startRequest = new StartQueryExecutionRequest
        {
            QueryString = query,
            QueryExecutionContext = new QueryExecutionContext { Database = "logs_db" },
            ResultConfiguration = new ResultConfiguration 
            { 
                OutputLocation = "s3://athena-results/"
            }
        };
        
        var response = await _athenaClient.StartQueryExecutionAsync(startRequest);
        
        Console.WriteLine($"Analyzing logs for {dateStr}");
        Console.WriteLine($"Query ID: {response.QueryExecutionId}");
    }
    
    // Calculate API response times by endpoint
    public async Task AnalyzeAPIPerformanceAsync(
        string dateStr)
    {
        var query = $@"
            SELECT 
                endpoint,
                method,
                COUNT(*) as request_count,
                AVG(response_time_ms) as avg_response_ms,
                MAX(response_time_ms) as max_response_ms,
                PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY response_time_ms) as p95_ms,
                SUM(CASE WHEN status_code >= 400 THEN 1 ELSE 0 END) as error_count
            FROM logs
            WHERE 
                year = '{dateStr.Substring(0, 4)}'
                AND month = '{dateStr.Substring(5, 2)}'
                AND day = '{dateStr.Substring(8, 2)}'
                AND service = 'api'
            GROUP BY endpoint, method
            ORDER BY avg_response_ms DESC
        ";
        
        var startRequest = new StartQueryExecutionRequest
        {
            QueryString = query,
            QueryExecutionContext = new QueryExecutionContext { Database = "logs_db" },
            ResultConfiguration = new ResultConfiguration 
            { 
                OutputLocation = "s3://athena-results/"
            }
        };
        
        var response = await _athenaClient.StartQueryExecutionAsync(startRequest);
        Console.WriteLine($"API Performance Analysis: {response.QueryExecutionId}");
    }
}

// Usage:
var analyticsService = new LogAnalyticsService(athenaClient);

// Analyze API performance
await analyticsService.AnalyzeAPIPerformanceAsync("2024-03-13");

⚠️ Common Athena Mistakes & Solutions

Mistake 1: No Partitioning

❌ Wrong: All data in one folder

s3://bucket/all_users.csv (100 GB)
Query scans 100% = $0.50 per query
Monthly cost: $15/day = $450/month

✅ Right: Partition by date

s3://bucket/year=2024/month=03/day=13/users.csv
Query scans only 1 day = 1/100 = $0.005
Monthly cost: $0.15/day = $4.50/month
Savings: 99%!

Mistake 2: Selecting All Columns

❌ Wrong: SELECT *

Scans all 100 columns
Data size: 100 GB scanned

✅ Right: SELECT specific columns

SELECT id, name, email (only 3 columns)
Data size: 3 GB scanned
Savings: 97%!

Mistake 3: Not Using Compression

❌ Wrong: CSV format

Uncompressed: 100 GB
Cost per query: $0.50

✅ Right: Parquet format

Compressed: 10 GB (10% size)
Cost per query: $0.05
Savings: 90%!

✅ Athena Best Practices Checklist

  • Always partition data (by date, region, customer)
  • Use Parquet or ORC format (not CSV)
  • Create Glue Catalog for automatic schema detection
  • Select only needed columns in queries
  • Add WHERE clauses to filter early
  • Use LIMIT for testing (before full scans)
  • Enable query result caching (ResultReuseByAgeConfiguration)
  • Set up workgroups for team management
  • Enable encryption for results in S3
  • Monitor data scanned vs results returned
  • Use Athena Preview features carefully
  • Archive old logs to Glacier (cheaper storage)
  • Set up CloudWatch alarms for query costs
  • Document table schemas in Glue Catalog
  • Periodically clean up old query results from S3
